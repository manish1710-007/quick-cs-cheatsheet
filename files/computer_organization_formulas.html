Number Conversions

Decimal to Other Bases
Repeated division by base, collect remainders (bottom-up)
📌 Decimal to Binary:
25₁₀ ÷ 2 = 12 R 1
12 ÷ 2 = 6 R 0
6 ÷ 2 = 3 R 0
3 ÷ 2 = 1 R 1
1 ÷ 2 = 0 R 1
Result: 11001₂
Other Bases to Decimal
Value = Σ(dᵢ × baseⁱ) where i = position from right (0-indexed)
📌 Binary to Decimal:
11001₂ = 1×2⁴ + 1×2³ + 0×2² + 0×2¹ + 1×2⁰
= 16 + 8 + 0 + 0 + 1 = 25₁₀
Binary ↔ Hexadecimal
Group binary in 4s, convert each group
📌 Example:
11010110₂ = 1101 0110 = D6₁₆
2F₁₆ = 0010 1111 = 101111₂
Binary ↔ Octal
Group binary in 3s, convert each group
📌 Example:
11010110₂ = 011 010 110 = 326₈
75₈ = 111 101 = 111101₂

Signed Number Representations

Method Positive Negative Range (n bits)
Sign-Magnitude 0 followed by magnitude 1 followed by magnitude -(2ⁿ⁻¹-1) to +(2ⁿ⁻¹-1)
One's Complement Same as binary Invert all bits -(2ⁿ⁻¹-1) to +(2ⁿ⁻¹-1)
Two's Complement Same as binary Invert all bits, add 1 -2ⁿ⁻¹ to +(2ⁿ⁻¹-1)
📌 Example: -5 in 8-bit
5 in binary: 00000101
One's complement: 11111010
Two's complement: 11111011 (add 1)
💡 Two's Complement Advantages:
• Single representation for zero
• Arithmetic is simpler (same hardware for signed/unsigned)
• Most commonly used in modern computers

Floating Point Representation

IEEE 754 Format
Value = (-1)ˢ × (1 + M) × 2^(E - bias)
Where:
• S = Sign bit (1 bit)
• E = Exponent (biased)
• M = Mantissa/Fraction (implicit leading 1)
Format Sign Exponent Mantissa Bias Total
Single Precision182312732 bits
Double Precision11152102364 bits
📌 Example: Convert 6.75 to IEEE 754 Single Precision
Step 1: 6.75 = 110.11₂ = 1.1011 × 2²
Step 2: Sign = 0 (positive)
Step 3: Exponent = 2 + 127 = 129 = 10000001₂
Step 4: Mantissa = 10110000000000000000000
Result: 0 10000001 10110000000000000000000

Special Values

• +0: S=0, E=0, M=0
• -0: S=1, E=0, M=0
• +∞: S=0, E=255, M=0
• -∞: S=1, E=255, M=0
• NaN: E=255, M≠0

Denormalized Numbers

When E=0, M≠0:
Value = (-1)ˢ × (0 + M) × 2^(-126)
Allows representation of very small numbers near zero.

🎯 CPU Organization

CPU Components

ALU (Arithmetic Logic Unit)

Performs arithmetic and logical operations:
• Arithmetic: ADD, SUB, MUL, DIV
• Logical: AND, OR, XOR, NOT
• Shift/Rotate: SHL, SHR, ROL, ROR

Control Unit

Directs CPU operations:
• Fetches instructions from memory
• Decodes instructions
• Generates control signals
• Coordinates data movement

Registers

Fast, small storage within CPU:
• PC: Program Counter
• IR: Instruction Register
• MAR: Memory Address Register
• MDR: Memory Data Register
• ACC: Accumulator
• General Purpose: R0-Rn

Flags/Status Register

• Z (Zero): Result is zero
• N (Negative): Result is negative
• C (Carry): Carry out from operation
• V (Overflow): Signed overflow
• P (Parity): Even parity of result

📜 Instruction Set Architecture (ISA)

Instruction Format

Three-Address Instruction
OPCODE Dest, Source1, Source2
Example: ADD R1, R2, R3 (R1 = R2 + R3)
Most flexible but requires more bits for addresses.
Two-Address Instruction
OPCODE Dest, Source
Example: ADD R1, R2 (R1 = R1 + R2)
Destination is also one source. Compact but overwrites destination.
One-Address Instruction
OPCODE Operand
Example: ADD R1 (ACC = ACC + R1)
Uses accumulator. Implicit second operand.
Zero-Address (Stack)
OPCODE (no operands)
Example: ADD (Top = Top + Second)
Uses stack-based evaluation. Operands implicit from stack.

Addressing Modes

Mode Effective Address Example
Immediate Operand in instruction MOV R1, #100
Direct (Absolute) EA = Address field MOV R1, 1000
Indirect EA = M[Address] MOV R1, @1000
Register EA = Register MOV R1, R2
Register Indirect EA = M[R] MOV R1, (R2)
Displacement EA = M[R + displacement] MOV R1, 100(R2)
Indexed EA = M[Index + base] MOV R1, array(Rx)
Based EA = M[Base + offset] MOV R1, offset(Rb)
PC-Relative EA = PC + offset JMP +100

Instruction Length Calculation

Instruction Size
Instruction Bits = Opcode bits + Σ(Address field bits × Number of addresses)
📌 Example:
CPU with 64 opcodes, 16 registers, 32-bit memory addresses
Three-address: 6 + 4 + 4 + 4 = 18 bits (register addresses)
One-address (memory): 6 + 32 = 38 bits

⚡ Pipelining

Basic Pipeline Concept

5-Stage RISC Pipeline

IF
Instruction

Instruction
Decode → EX
Execute → MEM
Memory
Access → WB
Write
Back
Each stage takes 1 clock cycle

Pipeline Performance

Speedup
Speedup = Time_non-pipelined / Time_pipelined
Ideal Speedup = n (number of stages)
📌 Example:
Non-pipelined: 5ns × 1000 instructions = 5000ns
Pipelined: 5ns + (1000-1) × 1ns = 1004ns
Speedup = 5000/1004 ≈ 4.98×
Pipeline Time
Total Time = k × τ + (n-1) × τ = (k + n - 1) × τ
k = Number of stages
n = Number of instructions
τ = Clock cycle time (time of slowest stage)
Throughput
Throughput = n / Total Time
Ideal Throughput = 1/τ (1 instruction per cycle)
Efficiency
Efficiency = (Useful time) / (Total time × Number of stages)
Efficiency = n / (k + n - 1)

Pipeline Hazards

🔴 Data Hazards
Occurs when an instruction depends on result of previous instruction
RAW (Read After Write)
Most common. True dependency.
Example:
ADD R1, R2, R3
SUB R4, R1, R5 ← Needs R1
WAR (Write After Read)
Anti-dependency. Rare in simple pipelines.
Example:
SUB R4, R1, R5
ADD R1, R2, R3 ← Writes R1
WAW (Write After Write)
Output dependency. Occurs in out-of-order execution.
✓ Solutions for Data Hazards:
1. Forwarding (Bypassing): Route result directly to waiting instruction
2. Stalling (Pipeline Bubbles): Insert NOPs until data ready
3. Code Scheduling: Reorder instructions to avoid hazards
🟠 Control Hazards
Occurs when pipeline cannot determine next instruction (branches)
Branch Penalty
Penalty = Number of cycles wasted on misprediction
Average Branch Cost = Penalty × Misprediction Rate
Solutions
• Branch Prediction: Guess branch outcome
• Delayed Branch: Execute useful instruction after branch
• Prefetching: Fetch both possible paths
🟡 Structural Hazards
Occurs when multiple instructions need same hardware resource
Example: Two instructions trying to access memory simultaneously.
Solution: Duplicate resources (separate instruction/data caches)

Speedup with Stalls

Actual Speedup
Actual Speedup = n × k / (k + n - 1 + S)
S = Number of stall cycles
n = Number of instructions
k = Number of pipeline stages
📌 Example:
5-stage pipeline, 100 instructions, 20 stall cycles
Ideal time: 5 + 99 = 104 cycles
Actual time: 104 + 20 = 124 cycles
Speedup = (100 × 5) / 124 = 4.03×

Average Memory Access Time

AMAT Formula
AMAT = Hit Time + Miss Rate × Miss Penalty
With Multiple Levels:
AMAT = H₁ × t₁ + (1-H₁) × [t₁ + H₂ × t₂ + (1-H₂) × (t₂ + t₃)]
H = Hit ratio, t = access time
Miss Rate = 1 - Hit Rate
📌 Example:
L1: Hit rate 95%, access time 1ns
L2: Hit rate 90%, access time 10ns
Main memory: access time 100ns

AMAT = 0.95(1) + 0.05[1 + 0.90(10) + 0.10(10+100)]
AMAT = 0.95 + 0.05[1 + 9 + 11] = 0.95 + 1.05 = 2ns

Cache Organization

Direct Mapped Cache

Block maps to exactly one cache line
Line = (Block Address) mod (Number of Lines)
Address Structure:
| Tag | Line/Index | Offset |

• Fast, simple hardware
• High conflict misses

Fully Associative Cache

Block can go anywhere in cache
Search all entries in parallel
Address Structure:
| Tag | Offset |

• Lowest miss rate
• Expensive, slow search

Set Associative Cache

Block maps to one set, can go in any line within set
Set = (Block Address) mod (Number of Sets)
Address Structure:
| Tag | Set Index | Offset |

• N-way set associative: N lines per set
• Balance between direct mapped and fully associative
• 2-way, 4-way, 8-way common

Cache Formulas

Number of Cache Lines
Number of Lines = Cache Size / Block Size
Number of Sets (Set Associative)
Number of Sets = Cache Size / (Block Size × Associativity)
Offset Bits
Offset bits = log₂(Block Size)
Index Bits
Index bits = log₂(Number of Sets or Lines)
Tag Bits
Tag bits = Address bits - Index bits - Offset bits
📌 Example:
32-bit address, 64KB cache, 32B blocks, 4-way associative
• Cache lines = 64KB / 32B = 2048 lines
• Sets = 2048 / 4 = 512 sets
• Offset bits = log₂(32) = 5
• Index bits = log₂(512) = 9
• Tag bits = 32 - 9 - 5 = 18 bits

Cache Miss Types

Type Cause Solution
Compulsory (Cold) First access to block Larger blocks, prefetching
Capacity Cache too small Larger cache
Conflict Multiple blocks map to same set Higher associativity

Cache Write Policies

Write Through

• Write to cache AND memory
• Always consistent
• Higher memory traffic
• No dirty bit needed

Write Back

• Write only to cache
• Update memory on eviction
• Lower memory traffic
• Needs dirty bit

Write Allocate

On write miss: bring block to cache, then write
Used with write-back

No Write Allocate

On write miss: write directly to memory
Used with write-through

Virtual Memory

Address Translation
Physical Address = Frame Number × Page Size + Offset
Virtual Address = Page Number × Page Size + Offset
Page Number bits = log₂(Number of Pages)
Offset bits = log₂(Page Size)

Page Table Entry (PTE)

Contains for each page:
• Frame Number: Physical location
• Valid Bit: Page in memory
• Dirty Bit: Page modified
• Reference Bit: Page accessed
• Protection: R/W/X permissions

Page Table Size

Page Table Size = Number of Pages × Entry Size
📌 Example:
32-bit VA, 4KB pages, 4B entries
Pages = 2³² / 2¹² = 2²⁰ = 1M
Page Table Size = 1M × 4B = 4MB

TLB (Translation Lookaside Buffer)

Effective Access Time = TLB Hit Rate × (TLB + Memory) + TLB Miss Rate × (TLB + Page Table + Memory)
Cache for page table entries. Small, fully associative cache.
Typical: 64-256 entries, ~95-99% hit rate

Page Replacement Algorithms

Algorithm Description Optimality
OPT/MIN Replace page used farthest in future Optimal (theoretical)
FIFO Replace oldest page Simple, may have Belady's anomaly
LRU Replace least recently used Good approximation of OPT
Clock LRU approximation with reference bit Practical implementation

🔌 I/O Organization

I/O Control Methods

Method CPU Involvement Speed Hardware
Programmed I/O Polls status continuously Slow Simple
Interrupt-Driven I/O Interrupted when ready Medium Moderate
DMA Only setup and completion Fast DMA Controller
Channel I/O Minimal Fastest I/O Processor

Programmed I/O

Polling Process

CPU reads device status register
Check if device is ready
If not ready, loop back to step 1
If ready, read/write data
Time wasted = Polling loops × Loop time
⚠️ Disadvantage:
CPU wastes cycles waiting for I/O device.

Interrupt-Driven I/O

Interrupt Process

CPU issues I/O command
CPU continues other work
Device sends interrupt when done
CPU saves context, jumps to ISR
ISR handles I/O, restores context

Interrupt Overhead

Overhead = Context Save + ISR Execution + Context Restore
Typically 2-10 microseconds per interrupt.

DMA (Direct Memory Access)

DMA Transfer
Transfer Time = Data Size / Transfer Rate
CPU Overhead = Setup Time + Post-processing Time

DMA Process

CPU programs DMA: source, dest, count
DMA requests bus (BR signal)
CPU grants bus (BG signal)
DMA transfers data directly
DMA sends interrupt when done

DMA Modes

Burst Mode: Transfer entire block at once
Cycle Stealing: Transfer one word, release bus
Transparent: Transfer during CPU cache miss
DMA Efficiency
Efficiency = (Data Size / Transfer Rate) / [(Data Size / Transfer Rate) + CPU Overhead]
📌 Example:
Transfer 1MB, Rate = 100MB/s, Overhead = 10μs
Transfer time = 1MB / 100MB/s = 10ms
CPU utilization = 10μs / 10ms = 0.1% overhead

Bus Arbitration

Daisy Chain

• Devices connected in series
• Priority by physical position
• Simple but unfair (farther devices starve)
• Single point of failure

Centralized Parallel

• All devices connect to arbitrator
• Parallel request lines
• Fair arbitration possible
• More complex wiring

Distributed Self-Selection

• Each device decides if it wins
• Based on priority scheme
• No central arbitrator

Distributed Collision Detection

• Devices detect conflicts
• Backoff and retry
• Used in Ethernet

📊 Performance Metrics

CPU Performance

CPU Time
CPU Time = Instructions × CPI × Clock Cycle Time
CPU Time = (Instructions × CPI) / Clock Rate
CPI = Cycles Per Instruction
Clock Rate = 1 / Clock Cycle Time
MIPS (Million Instructions Per Second)
MIPS = Instruction Count / (Execution Time × 10⁶)
MIPS = Clock Rate / (CPI × 10⁶)
Clock Rate
Clock Rate (Hz) = 1 / Cycle Time (seconds)
📌 Example:
Cycle time = 0.5ns = 0.5 × 10⁻⁹ s
Clock Rate = 1 / (0.5 × 10⁻⁹) = 2 GHz
Overall CPI
CPI = Σ(CPIᵢ × Frequencyᵢ)
CPIᵢ = CPI for instruction type i
Frequencyᵢ = Fraction of instructions of type i

Amdahl's Law

Speedup Formula
Speedup = 1 / [(1 - f) + f/s]
f = Fraction of execution time improved
s = Speedup of that fraction
📌 Example:
60% of time can be improved with 10× speedup
Speedup = 1 / [(1 - 0.6) + 0.6/10]
Speedup = 1 / [0.4 + 0.06] = 1 / 0.46 ≈ 2.17×
💡 Key Insight:
Maximum speedup when f=1 is s, but if f<1, speedup < s. Even infinite improvement to part of code has limited effect.

Performance Comparison

Relative Performance
Performance Ratio = Execution Time_B / Execution Time_A
"A is n times faster than B" means Time_B / Time_A = n
Average CPI Calculation
📌 Example:
InstructionFrequencyCPI
ALU50%1
Load/Store30%2
Branch20%3
CPI = 0.5×1 + 0.3×2 + 0.2×3
CPI = 0.5 + 0.6 + 0.6 = 1.7
FLOPS (Floating Point Operations/sec)
FLOPS = Number of FP Operations / Execution Time
Used for scientific computing benchmarks. Common units: MFLOPS, GFLOPS, TFLOPS.

Overhead

Cache Coherence

Snooping Protocol

Each cache monitors bus for memory operations.
MESI States:
• Modified - Only copy, dirty
• Exclusive - Only copy, clean
• Shared - Multiple copies
• Invalid - Not valid

Directory-Based

Central directory tracks which caches have each block.
• Scales better for large systems
• Higher latency for small systems
