📐 Set Theory

Basic Set Operations

Set Operations
A ∪ B = {x : x ∈ A or x ∈ B}
A ∩ B = {x : x ∈ A and x ∈ B}
A - B = {x : x ∈ A and x ∉ B}
A' = U - A (Complement)
A ⊕ B = (A - B) ∪ (B - A) (Symmetric Diff)
Cardinality Formulas
|A ∪ B| = |A| + |B| - |A ∩ B|
|A ∪ B ∪ C| = |A| + |B| + |C| - |A ∩ B| - |B ∩ C| - |A ∩ C| + |A ∩ B ∩ C|
Inclusion-Exclusion Principle: General formula for n sets with alternating sums
Power Set
|P(A)| = 2^n where n = |A|
The power set P(A) is the set of all subsets of A, including the empty set and A itself.
📌 Example:
If A = {1, 2}, then P(A) = {∅, {1}, {2}, {1,2}}
|P(A)| = 2² = 4 subsets

Set Identities (Laws)

Fundamental Laws

A ∪ ∅ = A (Identity)
A ∩ U = A (Identity)
A ∪ A' = U (Complement)
A ∩ A' = ∅ (Complement)
A ∪ U = U (Domination)
A ∩ ∅ = ∅ (Domination)
A ∪ A = A (Idempotent)
A ∩ A = A (Idempotent)

De Morgan's Laws

(A ∪ B)' = A' ∩ B'
(A ∩ B)' = A' ∪ B'
The complement of a union is the intersection of complements, and vice versa.

Associative Laws

(A ∪ B) ∪ C = A ∪ (B ∪ C)
(A ∩ B) ∩ C = A ∩ (B ∩ C)

Distributive Laws

A ∪ (B ∩ C) = (A ∪ B) ∩ (A ∪ C)
A ∩ (B ∪ C) = (A ∩ B) ∪ (A ∩ C)

Commutative Laws

A ∪ B = B ∪ A
A ∩ B = B ∩ A

Absorption Laws

A ∪ (A ∩ B) = A
A ∩ (A ∪ B) = A

Cartesian Product

Definition & Cardinality
A × B = {(a, b) : a ∈ A and b ∈ B}
|A × B| = |A| × |B|
📌 Example:
A = {1, 2}, B = {a, b}
A × B = {(1,a), (1,b), (2,a), (2,b)}
|A × B| = 2 × 2 = 4 ordered pairs

Implication Equivalences

Related Statements

Original: p → q
Converse: q → p
Inverse: ¬p → ¬q
Contrapositive: ¬q → ¬p
💡 Key Fact:
A statement and its contrapositive are logically equivalent!
p → q ≡ ¬q → ¬p

Important Equivalences

p → q ≡ ¬p ∨ q
p → q ≡ ¬q → ¬p
¬(p → q) ≡ p ∧ ¬q
p ↔ q ≡ (p → q) ∧ (q → p)
p ↔ q ≡ (p ∧ q) ∨ (¬p ∧ ¬q)

Logical Equivalences

Identity Laws

p ∧ T ≡ p
p ∨ F ≡ p

Domination Laws

p ∨ T ≡ T
p ∧ F ≡ F

Idempotent Laws

p ∨ p ≡ p
p ∧ p ≡ p

Double Negation

¬(¬p) ≡ p

De Morgan's Laws (Logic)

¬(p ∧ q) ≡ ¬p ∨ ¬q
¬(p ∨ q) ≡ ¬p ∧ ¬q

Absorption Laws

p ∨ (p ∧ q) ≡ p
p ∧ (p ∨ q) ≡ p

Associative Laws

(p ∨ q) ∨ r ≡ p ∨ (q ∨ r)
(p ∧ q) ∧ r ≡ p ∧ (q ∧ r)

Distributive Laws

p ∨ (q ∧ r) ≡ (p ∨ q) ∧ (p ∨ r)
p ∧ (q ∨ r) ≡ (p ∧ q) ∨ (p ∧ r)

Quantifiers

Types of Quantifiers

∀x P(x) — "For all x, P(x)"
∃x P(x) — "There exists x such that P(x)"
∃!x P(x) — "There exists exactly one x"

Negation of Quantifiers

¬(∀x P(x)) ≡ ∃x ¬P(x)
¬(∃x P(x)) ≡ ∀x ¬P(x)
💡 Remember:
"Not all" = "There exists some not"
"Not exists" = "All do not"

Normal Forms

DNF (Disjunctive Normal Form)

DNF = (term₁) ∨ (term₂) ∨ ... ∨ (termₙ)
OR of ANDs (Sum of Products). Each term is a conjunction of literals.
📌 Example:
(p ∧ q) ∨ (¬p ∧ r) ∨ (q ∧ ¬r)

CNF (Conjunctive Normal Form)

CNF = (clause₁) ∧ (clause₂) ∧ ... ∧ (clauseₙ)
AND of ORs (Product of Sums). Each clause is a disjunction of literals.
📌 Example:
(p ∨ q) ∧ (¬p ∨ r) ∧ (q ∨ ¬r)

🔗 Relations

Properties of Relations

Property Definition Formula
Reflexive Every element related to itself ∀a ∈ A: (a, a) ∈ R
Symmetric If aRb then bRa ∀a,b: (a,b) ∈ R → (b,a) ∈ R
Antisymmetric If aRb and bRa, then a=b (a,b) ∈ R ∧ (b,a) ∈ R → a=b
Transitive If aRb and bRc, then aRc (a,b) ∈ R ∧ (b,c) ∈ R → (a,c) ∈ R

Types of Relations

Equivalence Relation

Equivalence = Reflexive ∧ Symmetric ∧ Transitive
Partitions the set into disjoint equivalence classes.
📌 Example:
"Has the same remainder when divided by 3" is an equivalence relation on integers.

Partial Order

Partial Order = Reflexive ∧ Antisymmetric ∧ Transitive
Denoted by (A, ≤). Creates a poset (partially ordered set).
📌 Example:
"Is a subset of" (⊆) on a power set is a partial order.

Total Order

Total Order = Partial Order + Comparability
∀a,b: either aRb or bRa
Every pair of elements is comparable.

Equivalence Class

[a] = {x ∈ A : (a, x) ∈ R}
Set of all elements equivalent to a under relation R.

Relation Operations

Composition

R₁ ∘ R₂ = {(a,c) : ∃b, (a,b) ∈ R₂ ∧ (b,c) ∈ R₁}
📌 Example:
If R₁ = {(1,2), (2,3)} and R₂ = {(1,1), (1,2)}
R₁ ∘ R₂ = {(1,2), (1,3)}

Inverse

R⁻¹ = {(b,a) : (a,b) ∈ R}
Reverse all ordered pairs in the relation.

Reflexive Closure

R' = R ∪ {(a,a) : a ∈ A}
Add all self-loops to make R reflexive.

Symmetric Closure

R' = R ∪ R⁻¹
Add all inverse pairs to make R symmetric.

Transitive Closure

R* = R ∪ R² ∪ R³ ∪ ... ∪ Rⁿ
Where Rⁱ = R ∘ R ∘ ... ∘ R (i times)
The transitive closure includes all pairs reachable through any number of "hops".
💡 Warshall's Algorithm:
Efficient O(n³) algorithm to compute transitive closure using matrix operations.

📊 Functions

Types of Functions

Type Condition Formula
Injective (One-to-One) Different inputs → different outputs f(a) = f(b) → a = b
Surjective (Onto) Every element in codomain is mapped ∀b ∈ B, ∃a ∈ A: f(a) = b
Bijective Both injective and surjective One-to-one correspondence

Function Formulas

Composition

(f ∘ g)(x) = f(g(x))
Apply g first, then apply f to the result.
⚠️ Note:
Composition is associative but NOT commutative.
(f ∘ g) ∘ h = f ∘ (g ∘ h) but f ∘ g ≠ g ∘ f

Inverse Function

f⁻¹(f(x)) = x and f(f⁻¹(y)) = y
Only bijective functions have inverses.
📌 Example:
If f(x) = 2x + 3, then f⁻¹(y) = (y - 3) / 2

Number of Functions

Total Functions: A → B
|B|^|A|
Each of |A| elements has |B| choices
Injective Functions: A → B (|A| ≤ |B|)
P(|B|, |A|) = |B|! / (|B| - |A|)!
Permutation - no two elements map to same value
Bijective Functions: A → B (|A| = |B|)
|A|! = n!

🎲 Combinatorics

Basic Counting Principles

Product Rule

If task 1 has n₁ ways AND task 2 has n₂ ways
Total = n₁ × n₂
📌 Example:
5 shirts and 3 pants → 5 × 3 = 15 outfits

Sum Rule

If task 1 has n₁ ways OR task 2 has n₂ ways
Total = n₁ + n₂ (mutually exclusive)
📌 Example:
Choose from 3 math books OR 4 physics books → 3 + 4 = 7 choices

Permutations

Permutation (Arrangement)
P(n, r) = n! / (n - r)!
Number of ways to arrange r items from n distinct items (order matters).
Where: n = total items, r = items to arrange
📌 Example:
Arrange 3 people from 5: P(5, 3) = 5! / 2! = 60 ways

Permutation with Repetition

P(n; n₁, n₂, ..., nₖ) = n! / (n₁! × n₂! × ... × nₖ!)
Arranging n items where nᵢ are identical of type i.
📌 Example:
Arrange letters in "MISSISSIPPI":
11! / (1! × 4! × 4! × 2!) = 34,650

Circular Permutation

Circular(n) = (n - 1)!
If flipping allowed: (n - 1)! / 2
Arranging n distinct items in a circle (rotations considered same).
📌 Example:
5 people around a table: (5-1)! = 24 ways

Combinations

Combination (Selection)
C(n, r) = n! / (r! × (n - r)!) = (n choose r)
Number of ways to select r items from n distinct items (order does NOT matter).
📌 Example:
Choose 3 people from 5: C(5, 3) = 5! / (3! × 2!) = 10 ways

Combination Properties

C(n, r) = C(n, n - r)
C(n, 0) = C(n, n) = 1
C(n, 1) = n
C(n, r) = C(n-1, r-1) + C(n-1, r)
Pascal's Identity: Each entry is sum of two entries above it.

Combination with Repetition

C(n + r - 1, r) = (n + r - 1)! / (r! × (n - 1)!)
Select r items from n types (unlimited supply of each type).
📌 Example:
Choose 3 fruits from 5 types:
C(5 + 3 - 1, 3) = C(7, 3) = 35 ways

Binomial Theorem

Binomial Expansion
(x + y)ⁿ = Σ C(n, k) × x^(n-k) × y^k for k = 0 to n
= C(n,0)xⁿ + C(n,1)xⁿ⁻¹y + C(n,2)xⁿ⁻²y² + ... + C(n,n)yⁿ
📌 Example:
(x + y)³ = C(3,0)x³ + C(3,1)x²y + C(3,2)xy² + C(3,3)y³
= x³ + 3x²y + 3xy² + y³

Special Cases

Σ C(n, k) = 2ⁿ (for k = 0 to n)
Σ C(n, k) = 2ⁿ⁻¹ (for even k)
Σ (-1)^k × C(n, k) = 0
Sum of binomial coefficients equals 2ⁿ (total subsets).

Pascal's Triangle

1
1   1
1   2   1
1   3   3   1
1   4   6   4   1
1   5   10   10   5   1
Row n contains C(n, 0), C(n, 1), ..., C(n, n)

Pigeonhole Principle

Basic Form
If n items are placed into m containers and n > m, then at least one container has ⌈n/m⌉ items
Generalized Form
If n items are placed into k boxes, then at least one box contains at least ⌈n/k⌉ items
📌 Example:
In any group of 367 people, at least 2 share the same birthday.
(366 possible birthdays including Feb 29)

🕸️ Graph Theory

Basic Graph Definitions

Term Definition Formula/Property
Order Number of vertices |V| = n
Size Number of edges |E| = m
Degree Number of edges incident to vertex deg(v)
Isolated Vertex Vertex with degree 0 deg(v) = 0
Pendant Vertex Vertex with degree 1 deg(v) = 1

Handshaking Lemma

Sum of Degrees
Σ deg(v) = 2|E|
The sum of degrees of all vertices equals twice the number of edges.
Corollary: Every graph has an even number of vertices with odd degree.
📌 Example:
A graph with 5 vertices having degrees 2, 3, 3, 4, 4:
Sum = 2 + 3 + 3 + 4 + 4 = 16 = 2 × 8 edges

Types of Graphs

Complete Graph Kₙ

|E| = n(n-1)/2 = C(n, 2)
Every pair of vertices connected by an edge.
Each vertex has degree (n-1)

Cycle Graph Cₙ

|E| = n
Vertices connected in a single cycle.
Each vertex has degree 2

Complete Bipartite Kₘ,ₙ

|E| = m × n
Two sets of vertices (m and n), all cross-edges present.

Wheel Graph Wₙ

|E| = 2(n-1)
Cycle Cₙ₋₁ with a central hub connected to all vertices.

Regular Graph

All vertices have same degree k
|E| = nk/2
k-regular graph: every vertex has degree k.

Bipartite Graph

Graph is bipartite ⟺ No odd cycle
Vertices can be divided into two sets with no edges within a set.

Graph Connectivity

Euler's Formula (Planar Graphs)

V - E + F = 2
Where: V = vertices, E = edges, F = faces
📌 Example:
For a cube: V = 8, E = 12, F = 6
8 - 12 + 6 = 2 ✓

Euler Path & Circuit

Euler Path: Exactly 2 vertices with odd degree
Euler Circuit: All vertices have even degree
Euler path visits every edge exactly once.
Euler circuit starts and ends at same vertex.

Hamiltonian Path & Circuit

Hamiltonian Path: Visits every vertex exactly once
Hamiltonian Circuit: Path returns to start
⚠️ Note:
No simple necessary and sufficient condition exists for Hamiltonian paths/circuits (NP-complete problem).

Dirac's Theorem

If deg(v) ≥ n/2 for all vertices, then G has Hamiltonian circuit
Sufficient (not necessary) condition for Hamiltonian circuit in simple graph with n ≥ 3.

Graph Coloring

Chromatic Number

χ(G) = minimum colors needed
Minimum number of colors to color vertices so adjacent vertices have different colors.
χ(Kₙ) = n
χ(Cₙ) = 2 if n even, 3 if n odd
χ(Bipartite) = 2

Chromatic Polynomial

P(G, k) = number of ways to color G with k colors
P(Kₙ, k) = k(k-1)(k-2)...(k-n+1)
P(T, k) = k(k-1)ⁿ⁻¹ for tree with n vertices

Shortest Path Algorithms

Dijkstra's Algorithm

Finds shortest path from source to all vertices in weighted graph (non-negative weights).
Time Complexity: O(V²) or O((V+E) log V) with priority queue

Floyd-Warshall Algorithm

D[i,j] = min(D[i,j], D[i,k] + D[k,j])
Finds all-pairs shortest paths.
Time Complexity: O(V³)

🌳 Trees

Tree Properties

Fundamental Properties
A tree with n vertices has (n - 1) edges
A connected graph with n vertices and (n-1) edges is a tree
A graph with no cycles and (n-1) edges is a tree
Equivalent Definitions:
• Connected and acyclic
• Connected with n-1 edges
• Acyclic with n-1 edges
• Unique path between any two vertices

Spanning Tree

A subgraph that is a tree and includes all vertices
Number of edges = n - 1
📌 Number of Spanning Trees:
Complete graph Kₙ: n^(n-2) (Cayley's formula)

Minimum Spanning Tree (MST)

Spanning tree with minimum total edge weight.
Algorithms:
• Kruskal's: O(E log E)
• Prim's: O(E log V)

Rooted Trees

Terminology

• Root: Designated vertex (top)
• Parent: Vertex directly above
• Child: Vertices directly below
• Leaf: Vertex with no children
• Height: Longest path from root to leaf
• Depth: Distance from root

Binary Tree Properties

Max nodes at level i = 2^i
Max nodes in tree of height h = 2^(h+1) - 1
Min height for n nodes = ⌊log₂ n⌋
Leaves = Internal nodes + 1 (full binary tree)

Full m-ary Tree

i = (n - 1) / m
l = ((m - 1)n + 1) / m
n = mi + 1
Where: i = internal nodes, l = leaves, n = total nodes

Tree Traversals

• Preorder: Root → Left → Right
• Inorder: Left → Root → Right
• Postorder: Left → Right → Root
• Level-order: BFS from root

🔢 Number Theory

Divisibility

Division Algorithm

a = q × b + r, where 0 ≤ r < b
Where: a = dividend, b = divisor, q = quotient, r = remainder

Divisibility Rules

• Divisible by 2: Last digit even
• Divisible by 3: Sum of digits divisible by 3
• Divisible by 5: Last digit 0 or 5
• Divisible by 9: Sum of digits divisible by 9
• Divisible by 11: Alternating sum divisible by 11

GCD and LCM

Euclidean Algorithm
gcd(a, b) = gcd(b, a mod b)
gcd(a, 0) = a
📌 Example:
gcd(48, 18) = gcd(18, 12) = gcd(12, 6) = gcd(6, 0) = 6

GCD Properties

gcd(a, b) × lcm(a, b) = a × b
gcd(a, gcd(b, c)) = gcd(gcd(a, b), c)
gcd(ka, kb) = k × gcd(a, b)

Extended Euclidean Algorithm

gcd(a, b) = ax + by
Finds integers x, y such that ax + by = gcd(a, b)

Modular Arithmetic

Basic Operations

(a + b) mod n = ((a mod n) + (b mod n)) mod n
(a × b) mod n = ((a mod n) × (b mod n)) mod n
(a - b) mod n = ((a mod n) - (b mod n) + n) mod n

Properties

a ≡ b (mod n) ⟺ n | (a - b)
If a ≡ b and c ≡ d, then a+c ≡ b+d (mod n)
If a ≡ b and c ≡ d, then a×c ≡ b×d (mod n)

Modular Inverse

a^(-1) mod n exists ⟺ gcd(a, n) = 1
a × a^(-1) ≡ 1 (mod n)
The modular inverse exists only when a and n are coprime.

Prime Numbers

Fundamental Theorem of Arithmetic

n = p₁^a₁ × p₂^a₂ × ... × pₖ^aₖ
Every integer > 1 can be uniquely expressed as a product of prime powers.

Number of Divisors

d(n) = (a₁ + 1)(a₂ + 1)...(aₖ + 1)
📌 Example:
360 = 2³ × 3² × 5¹
d(360) = (3+1)(2+1)(1+1) = 24 divisors

Sum of Divisors

σ(n) = Σ pᵢ^(aᵢ+1) - 1 / (pᵢ - 1)
📌 Example:
σ(12) = σ(2² × 3) = (2³-1)/(2-1) × (3²-1)/(3-1) = 7 × 4 = 28

Euler's Totient Function

φ(n) = n × Π(1 - 1/pᵢ) for all prime factors pᵢ
φ(p) = p - 1 for prime p
φ(pᵏ) = pᵏ - p^(k-1)
📌 Example:
φ(12) = 12 × (1 - 1/2) × (1 - 1/3) = 12 × 1/2 × 2/3 = 4

Fermat's Little Theorem

If p is prime and gcd(a, p) = 1, then a^(p-1) ≡ 1 (mod p)
For any prime p: a^p ≡ a (mod p)
📌 Example:
Find 3¹⁰⁰ mod 7:
By Fermat: 3⁶ ≡ 1 (mod 7)
3¹⁰⁰ = (3⁶)¹⁶ × 3⁴ ≡ 1 × 81 ≡ 4 (mod 7)

🔄 Recurrence Relations

Linear Homogeneous Recurrences

General Form
aₙ = c₁aₙ₋₁ + c₂aₙ₋₂ + ... + cₖaₙ₋ₖ
Solution depends on characteristic equation: r^k - c₁r^(k-1) - ... - cₖ = 0

Distinct Roots

aₙ = α₁r₁ⁿ + α₂r₂ⁿ + ... + αₖrₖⁿ
If characteristic equation has k distinct roots r₁, r₂, ..., rₖ
📌 Example: Fibonacci
Fₙ = Fₙ₋₁ + Fₙ₋₂
Characteristic: r² - r - 1 = 0
r = (1 ± √5)/2
Fₙ = (φⁿ - ψⁿ)/√5 where φ = (1+√5)/2

Repeated Roots

If r is root with multiplicity m:
aₙ = (α₀ + α₁n + ... + αₘ₋₁n^(m-1))rⁿ
📌 Example:
aₙ = 4aₙ₋₁ - 4aₙ₋₂
Characteristic: r² - 4r + 4 = 0
r = 2 (multiplicity 2)
aₙ = (α₀ + α₁n)2ⁿ

Non-Homogeneous Recurrences

Particular Solution Method
aₙ = Homogeneous Solution + Particular Solution
1. Solve the homogeneous part
2. Find a particular solution based on the non-homogeneous term
3. Add both solutions and apply initial conditions

Guessing Particular Solutions

Non-homogeneous term Try particular solution
Constant c Constant A
Polynomial of degree d Polynomial of degree d
c × rⁿ A × rⁿ (or An×rⁿ if r is a root)

Master Theorem

For T(n) = aT(n/b) + f(n) where a ≥ 1, b > 1
Compare f(n) with n^(log_b a):
Case Condition Result
1 f(n) = O(n^(log_b a - ε)) for some ε > 0 T(n) = Θ(n^(log_b a))
2 f(n) = Θ(n^(log_b a) × log^k n) T(n) = Θ(n^(log_b a) × log^(k+1) n)
3 f(n) = Ω(n^(log_b a + ε)) and af(n/b) ≤ cf(n) T(n) = Θ(f(n))
📌 Example: Merge Sort
T(n) = 2T(n/2) + n
a = 2, b = 2, f(n) = n
n^(log₂ 2) = n¹ = n, so f(n) = Θ(n)
Case 2: T(n) = Θ(n log n)

⊛ Group Theory Basics

Group Axioms

Property Definition Formula
Closure Result of operation is in the set ∀a,b ∈ G: a * b ∈ G
Associativity Grouping doesn't matter (a * b) * c = a * (b * c)
Identity Element that doesn't change others ∃e: a * e = e * a = a
Inverse Every element has an inverse ∀a ∃a⁻¹: a * a⁻¹ = e

Types of Groups

Abelian (Commutative) Group

∀a,b ∈ G: a * b = b * a
A group where the operation is commutative.

Cyclic Group

G = ⟨g⟩ = {g⁰, g¹, g², ..., g^(n-1)}
Generated by a single element. Order of generator divides |G|.

Subgroup

H ⊆ G is subgroup ⟺ closed under * and inverses
Lagrange's Theorem: |H| divides |G|

Order of Element

ord(a) = smallest k > 0: a^k = e
ord(a) divides |G|
