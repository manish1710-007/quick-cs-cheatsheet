📊 CPU Scheduling

Scheduling Metrics

Turnaround Time (TAT)
TAT = Completion Time - Arrival Time
TAT = Waiting Time + Burst Time
Total time from arrival to completion of process.
Waiting Time (WT)
WT = Turnaround Time - Burst Time
WT = Start Time - Arrival Time (for first execution)
Time spent waiting in the ready queue.
Response Time (RT)
RT = First Response Time - Arrival Time
RT = Time of first CPU allocation - Arrival Time
Time from submission to first response (important for interactive systems).
CPU Utilization
CPU Utilization = (CPU Busy Time / Total Time) × 100%
Percentage of time the CPU is executing processes (not idle).
Throughput
Throughput = Number of Processes / Total Time
Number of processes completed per unit time.
Average Metrics
Average TAT = Σ(TATᵢ) / n
Average WT = Σ(WTᵢ) / n
Where: n = number of processes

Scheduling Algorithms

FCFS (First Come First Serve)

Order: Sorted by Arrival Time (Non-preemptive)
Gantt Chart Order
P₁ → P₂ → P₃ → ... (by arrival time)
Calculations
Completion Time = Previous Completion + Burst Time
First process: CT = Arrival + Burst
📌 Example:
P1: AT=0, BT=5 → CT=5, TAT=5, WT=0
P2: AT=1, BT=3 → CT=8, TAT=7, WT=4
P3: AT=2, BT=8 → CT=16, TAT=14, WT=6
⚠️ Convoy Effect:
Short processes wait behind long processes, increasing average waiting time significantly.

SJF (Shortest Job First) - Non-preemptive

Order: Process with shortest Burst Time first (among arrived processes)
Select: argmin(Burst Time) among ready processes
📌 Example:
At time 0: P1(AT=0, BT=7), P2(AT=2, BT=4), P3(AT=3, BT=1), P4(AT=4, BT=4)
Order: P1(0-7) → P3(7-8) → P2(8-12) → P4(12-16)
💡 Note:
SJF gives minimum average waiting time but requires knowing burst times in advance.

SRTF (Shortest Remaining Time First) - Preemptive

At any time, run process with minimum remaining burst time
Preempt when new process arrives with BT < Remaining BT of current
When new process arrives, compare its BT with current process's remaining time
If new BT < remaining, preempt current process
Context switch to the new shorter process
📌 Example:
P1(AT=0, BT=8), P2(AT=1, BT=4)
At t=1: P1 remaining=7, P2 BT=4 → Preempt P1, run P2

Priority Scheduling

Order: Process with highest priority first (lower number = higher priority)
Non-preemptive
Run selected process to completion
Preemptive
Preempt if new process has higher priority
⚠️ Starvation Problem:
Low priority processes may never execute. Solution: Aging - gradually increase priority of waiting processes.

Round Robin (RR)

Time Quantum: q (fixed time slice)
Each process gets q time, then moves to end of ready queue
Number of Context Switches
Approximately: Σ⌈Burst Time / Quantum⌉
Turnaround Time
TAT = Completion Time - Arrival Time
📌 Example:
P1(BT=10), P2(BT=5), P3(BT=8), Quantum=2
Gantt: P1(2)→P2(2)→P3(2)→P1(2)→P2(2)→P3(2)→P1(2)→P3(2)→P1(2)→P1(2)
💡 Quantum Selection:
• Too small: Excessive context switches
• Too large: Behaves like FCFS
• Typical: 10-100 milliseconds

HRRN (Highest Response Ratio Next)

Response Ratio = (Waiting Time + Burst Time) / Burst Time
Response Ratio = 1 + (Waiting Time / Burst Time)
Select: Process with highest Response Ratio
Non-preemptive algorithm that prevents starvation by considering waiting time.
📌 Example:
At time t=10, Process A: BT=5, WT=5 → RR = 1 + 5/5 = 2
Process B: BT=2, WT=2 → RR = 1 + 2/2 = 2
Process C: BT=10, WT=0 → RR = 1 + 0/10 = 1

Algorithm Comparison

Algorithm Type Starvation Best For Avg WT
FCFS Non-preemptive No Batch systems High (Convoy)
SJF Non-preemptive Yes (long jobs) Batch systems Minimum
SRTF Preemptive Yes (long jobs) Batch systems Minimum
Priority Both Yes (low priority) Real-time Variable
Round Robin Preemptive No Time-sharing Medium
HRRN Non-preemptive No (Aging built-in) General purpose Low

Multi-Level Queue Scheduling

Queue Structure

• Queue 1 (Highest Priority): System processes
• Queue 2: Interactive processes
• Queue 3: Interactive editing
• Queue 4: Batch processes
• Queue 5 (Lowest): Student processes

Scheduling Between Queues

• Fixed Priority: Serve all in Q1, then Q2, etc.
• Time Slice: Each queue gets CPU time proportion
• Each queue can use different scheduling algorithm

Multi-Level Feedback Queue (MLFQ)

Processes can move between queues based on their CPU burst characteristics:
New processes enter highest priority queue
If process uses full time slice → demote to lower queue
If process blocks before time slice → keep or promote
Lower queues have longer time quanta
Aging: Promote long-waiting processes

🔒 Process Synchronization

Critical Section Problem

Critical Section Structure

do {
// Entry Section
// Critical Section (access shared resource)
// Exit Section
// Remainder Section
} while(true);

Requirements

Mutual Exclusion: Only one process in critical section at a time
Progress: If no process in CS and processes want to enter, selection can't be postponed
Bounded Waiting: Limit on how many times other processes can enter before you

Peterson's Solution

// Shared: flag[2], turn
flag[i] = true;
turn = j;
while(flag[j] && turn == j);
// Critical Section
flag[i] = false;
⚠️ Limitation:
May not work on modern CPUs due to instruction reordering. Requires memory barriers.

Synchronization Hardware

Test-and-Set

boolean TestAndSet(boolean *target) {
boolean rv = *target;
*target = true;
return rv;
}
Atomic instruction that tests and sets a value in one uninterruptible operation.

Compare-and-Swap (CAS)

int CompareAndSwap(int *value, int expected, int new_value) {
int temp = *value;
if (*value == expected)
*value = new_value;
return temp;
}

Semaphores

Semaphore Operations
wait(S): while(S <= 0); S--;
signal(S): S++;
Semaphore S is an integer variable accessed only through wait() and signal() operations.

Counting Semaphore

Initial value: n (resource count)
Controls access to a resource with multiple instances (e.g., n buffer slots).

Binary Semaphore (Mutex)

Initial value: 1
Provides mutual exclusion. Only one process can access critical section.

Classic Problems

Producer-Consumer
Semaphore empty = n; // empty slots
Semaphore full = 0; // full slots
Semaphore mutex = 1; // mutual exclusion
Producer: wait(empty), wait(mutex), produce, signal(mutex), signal(full)
Consumer: wait(full), wait(mutex), consume, signal(mutex), signal(empty)
Readers-Writers
Semaphore rw_mutex = 1;
Semaphore mutex = 1;
int read_count = 0;
Multiple readers can read simultaneously. Writers need exclusive access.
Dining Philosophers
Semaphore chopstick[5] = {1, 1, 1, 1, 1};
Each philosopher needs two chopsticks (semaphores) to eat. Risk of deadlock.

Monitors

Monitor Structure

monitor MonitorName {
// Shared variables
condition x, y; // Condition variables
// Procedures
procedure entry P1(...) { ... }
procedure entry P2(...) { ... }
// Initialization code
}
Condition Variable Operations
x.wait(): Block until x.signal()
x.signal(): Wake up one waiting process

💀 Deadlock

Necessary Conditions (All Required)

Condition Description Prevention Strategy
Mutual Exclusion Resource cannot be shared Use spooling for printers
Hold and Wait Hold resources while waiting for more Request all resources at once
No Preemption Cannot forcibly take resources Preempt resources if possible
Circular Wait Circular chain of waiting processes Order resources, request in order

Resource-Allocation Graph

Graph Components

• Circles (P): Processes
• Squares (R): Resource types
• Edge P→R: Process requesting resource
• Edge R→P: Resource assigned to process
• Dots in Square: Instances of resource
💡 Deadlock Detection:
If graph contains a cycle AND each resource has only one instance → Deadlock
If cycle AND multiple instances → May or may not be deadlock

Banker's Algorithm

Data Structures
Available[m] // Available resources of each type
Max[n][m] // Maximum demand of each process
Allocation[n][m] // Currently allocated resources
Need[n][m] // Remaining resource need
Need[i][j] = Max[i][j] - Allocation[i][j]

Safety Algorithm

Initialize: Work = Available, Finish[i] = false for all i
Find i such that: Finish[i] = false AND Need[i] ≤ Work
If found: Work = Work + Allocation[i], Finish[i] = true, goto step 2
If all Finish[i] = true → System in safe state
Safe state means there exists a sequence where all processes can complete.

Resource-Request Algorithm

Check: Request[i] ≤ Need[i] (otherwise error)
Check: Request[i] ≤ Available (otherwise wait)
Pretend allocate: Update matrices temporarily
Run safety algorithm: If safe, grant; else, restore and wait
📌 Example:
Available: A=3, B=3, C=2
Process | Max | Allocation | Need
P0 | 7,5,3 | 0,1,0 | 7,4,3
P1 | 3,2,2 | 2,0,0 | 1,2,2
P2 | 9,0,2 | 3,0,2 | 6,0,0
P3 | 2,2,2 | 2,1,1 | 0,1,1
P4 | 4,3,3 | 0,0,2 | 4,3,1
Safe Sequence: P1 → P3 → P4 → P0 → P2 or similar

Deadlock Detection & Recovery

Detection Algorithm

Similar to safety algorithm but uses "Available" and checks for deadlock rather than safe state.
If no process can proceed → Deadlock detected

Recovery Methods

Abort: Terminate deadlocked processes
Preempt: Take resources from some processes
Rollback: Return to safe checkpoint state

🧠 Memory Management

Address Binding

Stage When Description
Compile Time Compilation Absolute code, location known at compile time
Load Time Loading Relocatable code, compiler generates relocatable code
Execution Time Runtime Binding delayed until runtime (MMU needed)

Logical vs Physical Address

MMU Translation
Physical Address = Logical Address + Relocation Register
Memory Management Unit (MMU) converts logical addresses to physical addresses at runtime.
Address Space
Logical Address Space: Process view of memory
Physical Address Space: Actual memory locations

Contiguous Memory Allocation

Partitioning Schemes

• Fixed Partitioning: Equal or varying size partitions, allocated as whole
• Variable Partitioning: Holes of various sizes scattered

Allocation Algorithms

First Fit: First hole that is big enough
Best Fit: Smallest hole that is big enough
Worst Fit: Largest hole (creates largest remaining)
Fragmentation
External Fragmentation
Total space exists but not contiguous. Solution: Compaction (move processes to create large free block).
Internal Fragmentation
Allocated memory is slightly larger than requested. Wasted space inside partitions.

Paging

Basic Formulas
Page Number = ⌊Logical Address / Page Size⌋
Page Offset = Logical Address mod Page Size
Number of Pages = Logical Address Space / Page Size
Number of Frames = Physical Address Space / Frame Size
Physical Address Calculation
Physical Address = Frame Number × Page Size + Offset
Or: PA = (Frame Number << Offset Bits) | Offset
📌 Example:
Logical Address: 8196, Page Size: 4KB
Page Number = 8196 / 4096 = 2
Offset = 8196 mod 4096 = 4
If Page 2 → Frame 5, PA = 5 × 4096 + 4 = 20484
Page Table Size
Page Table Size = Number of Pages × Entry Size
📌 Example:
32-bit system, 4KB pages, 4 bytes per entry
Pages = 2³² / 2¹² = 2²⁰ = 1M entries
Page Table Size = 1M × 4 = 4MB per process

Translation Lookaside Buffer (TLB)

Effective Access Time with TLB
EAT = Hit Ratio × (TLB Time + Memory Time)
+ (1 - Hit Ratio) × (TLB Time + 2 × Memory Time)
Simplified: EAT = TLB Time + Memory Time + (1 - Hit Ratio) × Memory Time
📌 Example:
TLB Access = 20 ns, Memory Access = 100 ns, Hit Ratio = 0.8
EAT = 0.8 × (20 + 100) + 0.2 × (20 + 200)
EAT = 96 + 44 = 140 ns

Multi-Level Paging

Two-Level Paging
Logical Address = p1 | p2 | d
p1: Outer page number, p2: Inner page number, d: Offset
Memory Accesses Required: 3 (outer table + inner table + data)
📌 Example: 32-bit address, 4KB pages, 4 bytes per entry
Offset = 12 bits
Remaining = 20 bits → Split into two 10-bit fields
Outer table: 2¹⁰ = 1024 entries
Inner table: 2¹⁰ = 1024 entries each

Segmentation

Segment Table Entry

Segment Table Entry = (Base, Limit)
Base: Starting physical address
Limit: Length of segment

Address Translation

If offset < limit:
Physical Address = Base + Offset
Else: Segmentation Fault

💿 Virtual Memory

Page Faults

Page Fault Handling
Check internal table for valid/invalid reference
If invalid, terminate. If valid but not in memory, page fault
Find free frame
Schedule disk operation to read page into frame
Update page table, restart instruction
Effective Access Time with Page Faults
EAT = (1 - p) × Memory Access Time + p × Page Fault Service Time
Where: p = Page fault rate (0 ≤ p ≤ 1)
📌 Example:
Memory Access = 100 ns, Page Fault Service = 8 ms = 8,000,000 ns
EAT with p = 0.001:
EAT = (1 - 0.001) × 100 + 0.001 × 8,000,000
EAT = 99.9 + 8000 = 8099.9 ns
Maximum Acceptable Page Fault Rate
p_max = (EAT_max - ma) / (Page Fault Service - ma)
📌 Example:
If acceptable EAT = 200 ns, ma = 100 ns, Page Fault = 10 ms
p_max = (200 - 100) / (10,000,000 - 100) ≈ 0.00001

Page Replacement Algorithms

FIFO (First In First Out)

Replace the page that has been in memory longest
Simple queue-based replacement. May suffer from Belady's Anomaly.
⚠️ Belady's Anomaly:
For some patterns, more frames can result in MORE page faults!

Optimal (OPT/MIN)

Replace page that will not be used for longest time in future
Theoretical optimum. Not implementable (requires future knowledge) but used for comparison.
💡 Key Point:
OPT has minimum possible page faults. Other algorithms compared against it.

LRU (Least Recently Used)

Replace page that has not been used for longest time
Counter Implementation
Each page has counter, updated on access
Replace page with smallest counter value
Stack Implementation
Move accessed page to top of stack
Replace page at bottom of stack

LRU Approximation

• Reference Bit: Set to 1 when page accessed
• Second Chance: Clock algorithm, give second chance if bit=1
• Enhanced Second Chance: Use (reference, modify) bits

Working Set Model

Working Set = {recently accessed pages in Δ time}
Total frames ≥ Σ Working Set sizes
If insufficient frames for working sets → Thrashing occurs

Thrashing

Definition & Detection
Thrashing occurs when a process spends more time paging than executing. CPU utilization drops sharply as the system is overwhelmed by page faults.
Causes:
• Degree of multiprogramming too high
• Insufficient frames per process
• Working sets not accommodated

Page Fault Frequency (PFF)

If page fault rate > upper bound → Allocate more frames
If page fault rate < lower bound → Reduce frames

Working Set Strategy

Monitor working set of each process
Ensure frames ≥ working set size
If not enough → Swap out entire process

Frame Allocation

Equal Allocation

Frames per process = Total Frames / Number of Processes

Proportional Allocation

Framesᵢ = (Sizeᵢ / ΣSizeⱼ) × Total Frames
💡 Minimum Frames:
Minimum frames required depends on instruction set architecture. Need enough to hold one instruction + operands.

💾 Disk Scheduling

Disk Service Time

Total Disk Service Time
Service Time = Seek Time + Rotational Latency + Transfer Time
Seek Time: Move arm to correct cylinder
Rotational Latency: Wait for sector (~half rotation average)
Transfer Time: Read/write the data

Scheduling Algorithms

FCFS

Total Movement = Σ |Positionᵢ - Positionᵢ₋₁|
Service requests in arrival order

SSTF

Next = argmin(|Current - Requestᵢ|)
Select closest request to current head position

SCAN (Elevator)

Move in one direction to end, then reverse
Movement = (End - Head) + (End - Min_Request)
Goes to disk end before reversing

C-SCAN (Circular)

One direction only, jump from end to start
Movement = (End - Head) + End + Max_Request
More uniform wait time than SCAN

LOOK

Like SCAN but only goes to extreme request
Movement = (Max_Request - Head) + (Max_Request - Min_Request)

C-LOOK

Like C-SCAN but only to extreme requests
Jump from max request to min request

Average Seek Time

Calculations
Average Seek Time = Total Head Movement / Number of Requests
Total Seek Time = Total Movement × Time per Cylinder

📁 File Systems

File Allocation Methods

Contiguous Allocation

Starting Block + Length
• Fast sequential and direct access
• Suffers from external fragmentation
• File size must be known in advance

Linked Allocation

Starting Block → Next Block → ... → NULL
• No external fragmentation
• Poor direct access performance
• Pointer overhead in each block

Indexed Allocation

Index Block → [Block Addresses]
Small Files
All pointers in one index block
Large Files
Linked index blocks
Or multilevel index

Free Space Management

Bit Vector

Bit[i] = 1 if block i is free, 0 if allocated
Space Required = Disk Size / Block Size bits
📌 Example:
16 GB disk, 4 KB blocks
Blocks = 16 GB / 4 KB = 4M blocks
Bitmap = 4M bits = 512 KB

Linked List

Head pointer → Free Block → Free Block → ...
Each free block contains pointer to next free block. No wasted space but traversing is slow.

Disk Block Size Trade-off

Factor Small Block Large Block
Internal Fragmentation Low High
Transfer Rate Lower Higher
I/O Overhead High (more I/O ops) Low
Random Access Better for small files Worse for small files

🔌 I/O Management

I/O Control Methods

Method Description CPU Usage
Programmed I/O CPU polls device status continuously Wasteful (busy waiting)
Interrupt-Driven CPU does other work, device interrupts when ready Efficient
DMA (Direct Memory Access) DMA controller transfers data, CPU interrupted only when done Most efficient for large transfers

I/O Buffering

Single Buffer

One buffer between user process and device. While process uses data in buffer, device can read next block.

Double Buffer

Two buffers alternate. Process uses one while device fills the other. Better throughput.

Circular Buffer

Multiple buffers in a ring. Accommodates speed differences between producer and consumer.

DMA Transfer

DMA Operation
CPU sets up DMA: source, destination, count
DMA controller transfers data without CPU involvement
DMA interrupts CPU when transfer complete
Transfer Time = Data Size / Transfer Rate
CPU free during transfer. Interrupt only at start and end.

🔐 Protection & Security

Access Matrix

Access Matrix Model
Access[Domain, Object] = {set of access rights}
Rows: Domains (protection domains)
Columns: Objects (files, devices, etc.)
Entry: Access rights (read, write, execute, etc.)

Access Control List (ACL)

Each object has list of (domain, rights) pairs
Column-wise decomposition of access matrix. Easy to answer: "Who can access this object?"

Capability List

Each domain has list of (object, rights) pairs
Row-wise decomposition of access matrix. Easy to answer: "What objects can this domain access?"

Protection Rings

x86 Protection Rings

• Ring 0: Kernel mode (highest privilege)
• Ring 1 & 2: Device drivers, system services
• Ring 3: User mode (lowest privilege)
💡 Privilege Check:
Process in ring i can only access resources at ring j where j ≥ i
