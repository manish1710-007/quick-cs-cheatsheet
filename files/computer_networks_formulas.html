⚡ Physical Layer

Transmission Basics

Bandwidth & Data Rate
Bit Rate = 2 × Bandwidth × log₂(L) (Nyquist)
Channel Capacity = Bandwidth × log₂(1 + S/N) (Shannon)
Where:
L = Number of signal levels
S/N = Signal-to-Noise Ratio (linear, not dB)
Signal-to-Noise Ratio (SNR)
SNR(dB) = 10 × log₁₀(Signal Power / Noise Power)
SNR(linear) = 10^(SNR_dB / 10)
📌 Example:
SNR = 30 dB → SNR(linear) = 10³ = 1000

Transmission Modes

Mode Description Example Channels
Simplex One-way communication only Keyboard to Monitor 1
Half-Duplex Two-way, one at a time Walkie-Talkie 1
Full-Duplex Two-way simultaneously Telephone, Mobile 2

Digital to Analog Modulation

Modulation Types

• ASK (Amplitude Shift Keying): Varies amplitude
• FSK (Frequency Shift Keying): Varies frequency
• PSK (Phase Shift Keying): Varies phase
• QAM (Quadrature Amplitude): Combines ASK + PSK

Baud Rate vs Bit Rate

Bit Rate = Baud Rate × log₂(L)
Where L = signal levels per baud
📌 Example:
2400 baud, 16-QAM (L=16):
Bit Rate = 2400 × log₂(16) = 2400 × 4 = 9600 bps

Multiplexing

FDM (Frequency Division)

• Multiple signals on different frequencies
• Used in: Radio, TV, Analog cellular
• Guard bands prevent interference

TDM (Time Division)

• Multiple signals in different time slots
• Synchronous: Fixed time slots
• Asynchronous: Dynamic allocation
• Used in: Digital telephone networks

WDM (Wavelength Division)

• Multiple light wavelengths on fiber
• DWDM: Dense WDM (many channels)
• Used in: Fiber optic networks

CDMA (Code Division)

• Each user has unique code
• All users share same frequency
• Used in: 3G cellular networks

🔗 Data Link Layer

Error Detection

Parity Check
• Simple Parity: 1 bit added (odd/even)
• Detects single-bit errors only
• Cannot detect even number of errors
Two-Dimensional Parity
• Arrange data in rows and columns
• Parity for each row and column
• Can detect and correct single-bit errors
Cyclic Redundancy Check (CRC)
FCS = Data ÷ Generator (mod-2 division)
Transmitted Data = Data + FCS
Append n zeros to data (n = degree of generator)
Divide by generator using XOR
Remainder = FCS (Frame Check Sequence)
At receiver: divide received data by generator
Zero remainder = No error detected
📌 Example:
Data: 1101, Generator: 1011 (CRC-3)
Append 3 zeros: 1101000
Divide: 1101000 ÷ 1011 → Remainder = 100
Transmitted: 1101100
CRC Type Generator Polynomial Application
CRC-8x⁸ + x² + x + 1ATM headers
CRC-16x¹⁶ + x¹⁵ + x² + 1USB, Modbus
CRC-32x³² + x²⁶ + x²³ + ... + 1Ethernet, ZIP

Error Correction

Hamming Distance

Hamming Distance = Number of bit positions where codewords differ
Minimum Distance d:
  • Detect up to (d-1) errors
  • Correct up to ⌊(d-1)/2⌋ errors

Hamming Code (7,4)

7 bits transmitted, 4 bits data, 3 bits parity
Positions: p₁ p₂ d₁ p₄ d₂ d₃ d₄
Parity bits at positions that are powers of 2 (1, 2, 4, ...)
Can detect and correct single-bit errors

Flow Control

Stop-and-Wait

Sender transmits one frame
Waits for ACK before sending next
Timeout triggers retransmission
Efficiency = 1 / (1 + 2a)
Where a = Propagation Time / Transmission Time

Sliding Window

Window Size: N frames can be sent without ACK
Go-Back-N: N ≤ 2ⁿ - 1
Selective Repeat: N ≤ 2ⁿ⁻¹
n = number of bits in sequence number
Efficiency Comparison
Stop-and-Wait: η = 1/(1 + 2a)
Go-Back-N: η = N/(1 + 2a) for N ≤ 1+2a, else 1
Selective Repeat: η = N/(1 + 2a) for N ≤ 1+2a, else 1

ARQ Protocols

Protocol Retransmission Window Size Buffer
Stop-and-Wait ARQ Single frame 1 1 frame
Go-Back-N ARQ All from error frame 2ⁿ - 1 None (receiver)
Selective Repeat ARQ Only error frame 2ⁿ⁻¹ Out-of-order frames

CSMA/CD (Ethernet)

Collision Detection
Minimum Frame Size = 2 × Propagation Delay × Data Rate
Slot Time = 2τ (round-trip propagation time)
For Ethernet: Slot time = 512 bit times
For 10 Mbps: 51.2 μs | For 100 Mbps: 5.12 μs

Binary Exponential Backoff

After collision, wait random time: r × slot time
r ∈ {0, 1, ..., 2ᵏ - 1}
k = min(number of collisions, 10)
Max 16 collisions, then abort

Subnetting

Subnet Calculations
Number of Subnets = 2ⁿ (n = subnet bits borrowed)
Hosts per Subnet = 2ʰ - 2 (h = host bits remaining)
Subnet Mask = Network bits + Subnet bits = 1s, rest 0s
Block Size = 256 - Subnet Mask (in octet being subnetted)
📌 Subnetting Example
Network: 192.168.1.0/24, need 4 subnets

Step 1: Borrow 2 bits: 2² = 4 subnets
Step 2: New mask: /26 = 255.255.255.192
Step 3: Block size: 256 - 192 = 64
Step 4: Hosts per subnet: 2⁶ - 2 = 62

Routing Algorithms

Distance Vector (RIP)

Bellman-Ford Algorithm
Dx(y) = min{c(x,v) + Dv(y)} for all neighbors v
• Shares entire routing table with neighbors
• Count-to-infinity problem
• Max hop count: 15 (RIP)
• Updates every 30 seconds

Link State (OSPF)

Dijkstra's Shortest Path Algorithm
• Each router knows complete topology
• Floods LSA (Link State Advertisements)
• No count-to-infinity
• Triggered updates

Path Vector (BGP)

• Inter-domain routing (between AS)
• Path information prevents loops
• Policy-based routing
• TCP port 179

Dijkstra's Algorithm

Initialize: Distance to source = 0, others = ∞
Mark all nodes unvisited
Select unvisited node with smallest distance
Update distances to neighbors
Mark current node visited, repeat from step 3

TCP Connection Management

Three-Way Handshake (Connection)

SYN: Client → Server (ISN = x)
SYN-ACK: Server → Client (ISN = y, ACK = x+1)
ACK: Client → Server (ACK = y+1)
Connection established

Four-Way Handshake (Termination)

FIN: Client → Server
ACK: Server → Client
FIN: Server → Client
ACK: Client → Server
Connection closed (TIME_WAIT = 2MSL)
💡 MSL (Maximum Segment Lifetime):
Maximum time a segment can exist in network. TIME_WAIT ensures old segments are gone before new connection uses same ports.

TCP Flow Control

Sliding Window
Window Size = Advertised Window (rwnd)
Bytes in flight ≤ min(cwnd, rwnd)
Sequence Number Space: 2³² = 4 GB
rwnd = Receiver's available buffer space
cwnd = Congestion window (sender's limit)

TCP Congestion Control

Phases

Slow Start:
  cwnd = 1 MSS, double each RTT
  cwnd = cwnd × 2 (exponential)
Congestion Avoidance:
  cwnd = cwnd + 1/cwnd (linear)
ssthresh: Slow start threshold

Loss Response

Timeout (Severe):
• ssthresh = cwnd/2
• cwnd = 1 MSS
• Go to Slow Start

3 Duplicate ACKs (Mild):
• ssthresh = cwnd/2
• cwnd = ssthresh + 3
• Fast Recovery

🔐 Network Security

Cryptography Basics

Symmetric Encryption

Same key for encryption & decryption
Algorithms: AES, DES, 3DES, RC4
Key Distribution: Problem - how to share key securely?
Speed: Fast (100-1000x faster than asymmetric)

Asymmetric Encryption

Public key encrypts, private key decrypts
Algorithms: RSA, ECC, Diffie-Hellman
Key Size: RSA 2048-bit ≈ AES 128-bit security
Use: Key exchange, digital signatures

Hash Functions

Properties
H(M) → Fixed-size hash from variable input
• Pre-image resistant: Hard to find M from H(M)
• Collision resistant: Hard to find M₁ ≠ M₂ with same hash
• Algorithms: SHA-256, SHA-3, MD5 (broken)
• Applications: Password storage, digital signatures, integrity

Digital Signatures

Signing Process

Hash the message: h = H(message)
Sign with private key: sig = Encrypt_private(h)
Send message + signature

Verification Process

Hash received message: h' = H(message)
Decrypt signature: h = Decrypt_public(sig)
Compare h and h'

SSL/TLS Handshake

TLS 1.2 Handshake Steps

ClientHello: Supported ciphers, TLS version, random
ServerHello: Selected cipher, certificate, random
Key Exchange: Client generates pre-master secret, encrypts with server's public key
Key Derivation: Both compute master secret, session keys
Finished: Verify handshake integrity
💡 Perfect Forward Secrecy:
Use ephemeral Diffie-Hellman (DHE/ECDHE) so compromise of private key doesn't reveal past session keys.

📊 Network Performance Metrics

Key Metrics

Bandwidth vs Throughput
Bandwidth = Maximum data rate (capacity)
Throughput = Actual data rate achieved
Efficiency = Throughput / Bandwidth × 100%
Latency (Delay)
Total Delay = Processing + Queuing + Transmission + Propagation
Transmission Delay = Packet Size / Bandwidth
Propagation Delay = Distance / Speed of Light in Medium
RTT (Round Trip Time)
RTT = Time for packet to travel to destination and back
Bandwidth-Delay Product = Bandwidth × RTT
BDP = Amount of data that can be "in flight"
Important for TCP window sizing
Goodput
Goodput = Useful Application Data / Total Time
Excludes protocol overhead, retransmissions, and headers.
Always ≤ Throughput ≤ Bandwidth

Performance Calculations

📌 Example Calculation
Given: 1 Gbps link, 20ms RTT, 1500 byte packets

Transmission Delay:
= 1500 × 8 bits / 10⁹ bps = 12 μs

Bandwidth-Delay Product:
= 10⁹ bps × 0.04s = 40 Mb = 5 MB

TCP Window needed: At least 5 MB to fill the pipe
