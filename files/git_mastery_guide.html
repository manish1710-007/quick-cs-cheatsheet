📌 Basic Commands

Initialize & Clone

# Initialize new repository git init # Clone existing repository git clone https://github.com/user/repo.git git clone git@github.com:user/repo.git # SSH git clone URL --branch main # Specific branch git clone URL --depth 1 # Shallow clone git clone URL --single-branch # One branch only

Status & Info

# Check status git status git status -s # Short format git status -sb # Short + branch info # Show commit history git log git log --oneline # One line per commit git log --oneline --graph --all # Visual graph git log -p -2 # Last 2 commits with diff git log --since="2 weeks ago" git log --author="John" git log --grep="bug" # Show file changes git diff # Unstaged changes git diff --staged # Staged changes git diff HEAD~3..HEAD # Compare commits git diff main..feature # Compare branches

Stage & Commit

# Stage files git add file.txt # Specific file git add . # All changes (current dir) git add -A # All changes (entire repo) git add -u # Only modified/deleted git add -p # Interactive staging # Commit git commit -m "Message" # Commit with message git commit -am "Message" # Add + commit (tracked only) git commit --amend # Amend last commit git commit --amend --no-edit # Keep same message # Unstage git restore --staged file # Unstage file git reset HEAD file # Older syntax

🌿 Branching

Branch Commands

# List branches git branch # Local branches git branch -a # All branches (local + remote) git branch -r # Remote branches only git branch -v # With last commit # Create branch git branch feature-name # Create from current git branch new-branch commit-hash # From specific commit # Switch branches git checkout branch-name git checkout -b new-branch # Create + switch git switch branch-name # Modern syntax git switch -c new-branch # Create + switch # Delete branch git branch -d branch-name # Safe delete (merged) git branch -D branch-name # Force delete git push origin --delete branch # Delete remote # Rename branch git branch -m old-name new-name git branch -m new-name # Rename current

Branch Visualization

main ●────●────●────●────● │ │ feature ●──●──● │ │ │ hotfix ●──────● │ (merged)

Merge Branches

# Merge into current branch git merge feature-branch # Merge options git merge --no-ff branch # Always create merge commit git merge --ff-only branch # Only if fast-forward git merge --squash branch # Squash into one commit # Abort merge (on conflicts) git merge --abort

Rebase

# Rebase current branch onto main git rebase main # Interactive rebase git rebase -i HEAD~3 # Last 3 commits git rebase -i base-commit # Continue/abort after conflict git rebase --continue git rebase --abort git rebase --skip # Skip current commit
⚠️ Rebase Warning: Never rebase commits that have been pushed to a shared repository. Rebase rewrites history and will cause problems for collaborators.

🌐 Remote Operations

Remote Management

# List remotes git remote -v # Add remote git remote add origin URL git remote add upstream URL # For forks # Update remote URL git remote set-url origin new-URL # Remove remote git remote remove origin # Show remote details git remote show origin

Fetch & Pull

# Fetch (download, don't merge) git fetch origin git fetch --all # All remotes git fetch --prune # Remove deleted remote branches # Pull (fetch + merge) git pull origin main git pull --rebase # Fetch + rebase (cleaner) git pull --rebase origin main # Set upstream tracking git branch -u origin/main # Set upstream git push -u origin branch # Push + set upstream

Push

# Push to remote git push origin main git push # With upstream set git push -f # Force push (dangerous!) git push --force-with-lease # Safer force push # Push all branches git push origin --all # Push tags git push origin --tags git push origin v1.0.0 # Specific tag # Delete remote branch git push origin --delete branch-name
💡 Fetch vs Pull:

git fetch: Downloads changes but doesn't modify your working directory. Safe to run anytime.

git pull: Downloads and merges in one step. Can cause merge conflicts.

📦 Stashing

💡 When to Stash: Save work-in-progress when you need to switch branches but don't want to commit incomplete work.
# Create stash git stash # Stash tracked files git stash -u # Include untracked files git stash -a # Include ignored files git stash push -m "WIP: feature" # With message # List stashes git stash list # Output: stash@{0}: On main: WIP: feature # View stash git stash show # Summary git stash show -p # Full diff git stash show stash@{2} # Specific stash # Apply stash git stash apply # Apply latest, keep in list git stash apply stash@{2} # Apply specific git stash pop # Apply + remove from list # Remove stash git stash drop # Remove latest git stash drop stash@{2} # Remove specific git stash clear # Remove all stashes # Create branch from stash git stash branch new-branch stash@{0}

⏪ Reset & Undo

Reset Types

git reset --soft HEAD~1 ┌─────────────────────────────────────────────────────────┐ │ Working Dir │ Staging │ Local Repo │ HEAD │ │ (unchanged) │ (unchanged) │ (moved back) │ ← │ └─────────────────────────────────────────────────────────┘ Changes are staged and ready to commit git reset --mixed HEAD~1 (default) ┌─────────────────────────────────────────────────────────┐ │ Working Dir │ Staging │ Local Repo │ HEAD │ │ (unchanged) │ (reset) │ (moved back) │ ← │ └─────────────────────────────────────────────────────────┘ Changes are unstaged (in working directory) git reset --hard HEAD~1 ┌─────────────────────────────────────────────────────────┐ │ Working Dir │ Staging │ Local Repo │ HEAD │ │ (reset) │ (reset) │ (moved back) │ ← │ └─────────────────────────────────────────────────────────┘ ALL CHANGES LOST! Use with extreme caution
# Undo last commit (keep changes) git reset --soft HEAD~1 # Undo last commit (unstage changes) git reset HEAD~1 # Same as --mixed # Undo last commit (discard all changes) git reset --hard HEAD~1 # Reset to specific commit git reset --hard abc123 # Undo git add git restore --staged file.txt git reset HEAD file.txt # Older syntax # Discard working directory changes git restore file.txt git checkout -- file.txt # Older syntax # Discard all local changes git restore . git checkout -- . # Older syntax
🚨 Danger Zone: git reset --hard permanently deletes all uncommitted changes. Always commit or stash important work before using it!

🔮 Reflog - Git's Time Machine

💡 Reflog tracks all HEAD movements: Even after reset --hard, you can recover commits using reflog. It's your safety net!
# View reflog git reflog # Output: # abc1234 HEAD@{0}: reset: moving to HEAD~1 # def5678 HEAD@{1}: commit: Add feature # ghi9012 HEAD@{2}: checkout: moving to main # Recover lost commit git reset --hard def5678 # Go back to that state # Recover deleted branch git reflog # Find the commit before branch was deleted git checkout -b recovered-branch abc1234 # View reflog for specific branch git reflog show main # Expire old reflog entries git reflog expire --expire=30.days.ago --all
📖 Reflog Retention:

By default, reflog entries are kept for 90 days. After that, they may be garbage collected. Use reflog immediately after accidental data loss.

🍒 Cherry Pick

# Cherry-pick specific commit git cherry-pick abc1234 # Cherry-pick range git cherry-pick abc1234..def5678 # Cherry-pick without committing git cherry-pick -n abc1234 # Cherry-pick with custom message git cherry-pick -e abc1234 # Abort cherry-pick git cherry-pick --abort # Continue after resolving conflicts git cherry-pick --continue

When to Use Cherry-Pick

  • Hotfix: Apply bug fix from develop to release branch
  • Backport: Bring feature to older version branch
  • Selective merge: Pick specific commits instead of merging entire branch
  • Recover: Bring back accidentally deleted commit

🔍 Git Bisect - Find Bugs Fast

💡 Binary Search for Bugs: Bisect uses binary search to find which commit introduced a bug. It's incredibly efficient for large histories.
# Start bisect git bisect start # Mark current commit as bad (has bug) git bisect bad # Mark known good commit git bisect good v1.0.0 # Or commit hash # Git will checkout middle commit # Test and mark: git bisect good # If bug not present git bisect bad # If bug present # Repeat until bisect finds the culprit # Git will show: "abc1234 is the first bad commit" # End bisect git bisect reset # Automated bisect with script git bisect start HEAD v1.0.0 git bisect run ./test-script.sh

⚔️ Merge Conflict Resolution

Conflict Markers

# Conflict markers in file: <<<<<<< HEAD Current branch changes ======= Incoming branch changes >>>>>>> feature-branch # Resolve by editing file, then: git add conflicted-file.txt git commit # Or git rebase --continue

Conflict Resolution Tools

# View conflicted files git status # Use merge tool git mergetool # Accept theirs (incoming changes) git checkout --theirs file.txt # Accept ours (current branch) git checkout --ours file.txt # Abort merge git merge --abort # Abort rebase git rebase --abort

Conflict Resolution Strategies

  • Keep both: Often needed for additions (imports, config)
  • Keep ours: When your changes supersede theirs
  • Keep theirs: When incoming changes are correct
  • Combine: Merge both sets of changes intelligently
  • Start fresh: Rewrite the feature with updated context

🐛 Debugging with Git

git blame - Line History

# Who changed this line? git blame file.txt git blame -L 10,20 file.txt # Lines 10-20 git blame -w file.txt # Ignore whitespace # Output format: # abc1234 (John Doe 2024-01-15 10:30:25 +0000 1) const x = 5;

git log - File History

# History of specific file git log --follow -p file.txt # Who changed this file? git log --format="%an" file.txt | sort | uniq -c # Show file at specific commit git show abc1234:file.txt # Find when line was added git log -S "function name" -p git log -G "regex" -p # Regex search

git grep - Search Code

# Search in tracked files git grep "function" git grep -n "pattern" # With line numbers git grep -c "pattern" # Count matches git grep "pattern" v1.0.0 # In specific tag git grep "pattern" HEAD~3 # In specific commit

🔧 Fixing Common Mistakes

1
Wrong commit message
git commit --amend -m "New message"
2
Forgot to add file
git add file && git commit --amend --no-edit
3
Committed to wrong branch
git cherry-pick or reset and recommit
4
Accidentally deleted branch
git reflog then recreate from commit
5
Reset --hard by mistake
git reflog to find lost commits
6
Pushed sensitive data
Use BFG Repo-Cleaner or git filter-branch
7
Merge instead of rebase

✅ Git Best Practices

📝 Commit Messages

  • Write clear, descriptive messages
  • Use imperative mood ("Add feature" not "Added")
  • First line: 50 chars max, summarize
  • Blank line, then detailed explanation
  • Reference issue numbers when applicable
  • Explain WHY, not just WHAT
# Good commit message format: Add user authentication feature - Implement JWT-based authentication - Add login/logout endpoints - Include password hashing with bcrypt Closes #123

🌿 Branching

  • Keep branches short-lived
  • Use descriptive branch names
  • Delete merged branches
  • One feature/fix per branch
  • Rebase before merging (cleaner history)
  • Never commit directly to main
# Branch naming conventions: feature/user-authentication bugfix/login-error hotfix/security-patch release/v2.0.0

🔄 Workflow

  • Pull before starting work
  • Commit early, commit often
  • Review your own diff before committing
  • Never force push to shared branches
  • Use .gitignore from the start
  • Tag releases consistently

🚫 Don't Do This

  • Commit sensitive data (passwords, keys)
  • Commit generated/binary files
  • Commit unrelated changes together
  • Rebase public/shared branches
  • Use --force without --with-lease
  • Leave merge conflicts unresolved
