Number Systems

Number System Conversions

Decimal to Binary

N₁₀ → Divide by 2, collect remainders (LSB to MSB)

Example:

25₁₀ → 25÷2=12 R1, 12÷2=6 R0, 6÷2=3 R0, 3÷2=1 R1, 1÷2=0 R1

Result: 11001₂

Binary to Decimal

N₂ = Σ dᵢ × 2ⁱ (i = 0 to n-1)

Example:

11001₂ = 1×2⁴ + 1×2³ + 0×2² + 0×2¹ + 1×2⁰

= 16 + 8 + 0 + 0 + 1 = 25₁₀

Decimal to Hexadecimal

N₁₀ → Divide by 16, collect remainders

Example:

255₁₀ → 255÷16=15 R15(F), 15÷16=0 R15(F)

Result: FF₁₆

Binary to Hexadecimal

Group 4 bits → Convert each group

Example:

11010110₂ = (1101)(0110) = D6₁₆

1101₂ = 13₁₀ = D, 0110₂ = 6₁₀ = 6

Two's Complement Conversion

-X = 2ⁿ - X = (1's complement) + 1

Example (8-bit):

+5 = 00000101₂

-5 = 11111010 + 1 = 11111011₂

Two's Complement to Decimal

If MSB=1: Value = -(2ⁿ - binary value)

Example:

11111011₂ (8-bit): MSB=1, so negative

Value = -(256 - 251) = -5₁₀

Boolean Algebra

Basic Laws & Theorems

Identity Laws

A + 0 = A
A · 1 = A

Null Laws

A + 1 = 1
A · 0 = 0

Idempotent Laws

A + A = A
A · A = A

Complement Laws

A + A' = 1
A · A' = 0

Involution Law

(A')' = A

Commutative Laws

A + B = B + A
A · B = B · A

Associative Laws

(A + B) + C = A + (B + C)
(A · B) · C = A · (B · C)

Distributive Laws

A · (B + C) = A·B + A·C
A + (B · C) = (A+B) · (A+C)

De Morgan's Theorems

First Theorem

(A + B)' = A' · B'

"The complement of OR equals AND of complements"

Second Theorem

(A · B)' = A' + B'

"The complement of AND equals OR of complements"

Example Application:

F = (A + B·C)' = A' · (B·C)' = A' · (B' + C')

Absorption & Consensus Theorems

Absorption Laws

A + A·B = A
A · (A + B) = A
A + A'·B = A + B
A · (A' + B) = A·B

Consensus Theorem

A·B + A'·C + B·C = A·B + A'·C
(A+B)·(A'+C)·(B+C) = (A+B)·(A'+C)

BC is redundant when A·B and A'·C are present

Canonical Forms

Sum of Products (SOP)

F = Σ m(minterms)

Product terms where function = 1

Example:

F(A,B,C) = Σm(1,3,5,7) = A'B'C + A'BC + AB'C + ABC

Product of Sums (POS)

F = Π M(maxterms)

Sum terms where function = 0

Example:

F(A,B,C) = ΠM(0,2,4,6) = (A+B+C)(A+B'+C)(A'+B+C)(A'+B'+C)

Minterm to Maxterm Conversion

Mᵢ = mᵢ'

If F = Σm(a,b,c) then F' = Σm(others)

F = ΠM(others)

Grouping Rules

1616 cells4Entire map = 1

Important Rules

  • Groups must be rectangular (or wrap-around)
  • Groups must contain 2ⁿ cells (power of 2)
  • Each group must be as large as possible
  • Each 1 must be covered at least once
  • Overlapping groups are allowed
  • Wrap-around edges are adjacent (top-bottom, left-right)

Counters

Asynchronous (Ripple) Counter

Modulus: M = 2ⁿ
Max frequency: f = 1/(n × t_pd)

Each FF triggered by previous FF output

Propagation delay accumulates

Synchronous Counter

Max frequency: f = 1/t_pd

All FFs clocked simultaneously

Faster but more complex

Mod-N Counter Design

n = ⌈log₂ N⌉ FFs required

Reset at count N (for mod-N)

Mod-6 Counter:

Need 3 FFs (2³ = 8 > 6)

Reset when count = 110 (6)

Johnson Counter (Twisted Ring)

Modulus: M = 2n (n FFs)

Q' of last FF fed back to D of first

Ring Counter

Modulus: M = n (n FFs)

Q of last FF fed back to D of first

FSM Design Procedure

  1. Understand the problem and define states
  2. Draw state diagram
  3. Create state table
  4. Assign binary codes to states
  5. Derive excitation equations
  6. Draw circuit diagram

State Reduction

Equivalent States: Two states are equivalent if they produce the same output for all inputs and transition to equivalent next states.

States can be merged if equivalent

State Encoding

Binary Encoding

FFs = ⌈log₂ n⌉ (n states)

Minimum flip-flops, more decoding logic

One-Hot Encoding

FFs = n (one FF per state)

More FFs, simpler next state logic

Only one bit = 1 at any time

Gray Code Encoding

Adjacent states differ by one bit

Reduces glitches, useful for transitions

Timing & Hazards

Timing Parameters

Flip-Flop Timing

Setup Time (tₛᵤ): Data must be stable before clock edge
Hold Time (tₕ): Data must be stable after clock edge
Clock-to-Q (tₐₒ): Output delay after clock edge

Maximum Frequency

Tₘᵢₙ ≥ tₐₒ + t_comb + tₛᵤ
fₘₐₓ = 1/Tₘᵢₙ

t_comb = combinational delay between FFs

Hold Time Constraint

tₕ ≤ tₐₒ + t_comb,min

Data must not arrive too quickly at next FF

Hazards

Types of Hazards

  • Static-1: Output should stay 1, but momentarily goes 0
  • Static-0: Output should stay 0, but momentarily goes 1
  • Dynamic: Output should transition, but has multiple transitions

Hazard Elimination

Add redundant terms to cover adjacent groups in K-map

Example:

F = AB + A'C (has static-1 hazard when B=1, C=1, A changes)

Add redundant term: F = AB + A'C + BC (Consensus theorem)
