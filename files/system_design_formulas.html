Reliability Calculations

MTBF = Total Uptime / Number of Failures
MTTR = Total Downtime / Number of Failures
Availability = MTBF / (MTBF + MTTR)

MTBF: Mean Time Between Failures

MTTR: Mean Time To Recovery

Throughput & Latency

Throughput = Requests / Second (RPS)
Latency = Time for single request
Little's Law: L = О» Г— W

L = average requests in system

О» = arrival rate, W = average wait time

Capacity Planning

Users = Daily Active Users Г— Requests/User/Day
Peak RPS = Average RPS Г— Peak Factor (2-3x)
Storage = Users Г— Data/User Г— Replication Factor
Bandwidth = (Requests/sec) Г— Avg Response Size

Sharding Challenges

  • Cross-shard queries: Joins across shards are expensive
  • Hot spots: Uneven data distribution
  • Consistency: Harder to maintain ACID across shards

    Pick 2 of 3 in distributed systems:

    CA: Traditional RDBMS (no partition tolerance)
    CP: MongoDB, HBase, Redis (consistent)
    AP: Cassandra, DynamoDB, CouchDB (available)

Consistency (C)

All nodes see same data at same time

Every read returns latest write or error

Trade-off: Higher latency, may be unavailable

Availability (A)

Every request receives a response

No guarantee it's the most recent data

Trade-off: May return stale data

Partition Tolerance (P)

System continues despite network failures

Network partitions are inevitable

Reality: Must choose P in distributed systems

Cache Hit Ratio

Hit Ratio = Cache Hits / Total Requests
Miss Rate = 1 - Hit Ratio
Avg Access Time = Hit Ratio Г— Cache Time + Miss Rate Г— DB Time

Example:

Cache time: 1ms, DB time: 100ms

Hit ratio: 90%

Avg time = 0.9 Г— 1 + 0.1 Г— 100 = 0.9 + 10 = 10.9ms
