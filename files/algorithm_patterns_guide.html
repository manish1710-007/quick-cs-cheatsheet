1 Two Pointers

📌 What is it?

Two pointers technique uses two variables to traverse data from different directions or at different speeds. Think of it like two people walking towards each other on a bridge!

Finding Pair with Target Sum in Sorted Array:

Array: [1, 2, 3, 4, 6, 8, 9, 11]  Target = 10

Step 1: left → [1]                    right → [11]
        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
        1   2   3   4   6   8   9   11
        ↑                               ↑
        left                           right
        Sum = 1 + 11 = 12 > 10 → move right left

Step 2: left → [1]                right → [9]
        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
        1   2   3   4   6   8   9   11
        ↑                           ↑
        left                       right
        Sum = 1 + 9 = 10 = 10 → ✓ FOUND!

✅ Found pair: (1, 9) at indices 0 and 6

🎯 When to Use

  • Sorted arrays - finding pairs, triplets
  • Palindromes - check from both ends
  • Container with most water - maximize area
  • Removing duplicates from sorted array
Time: O(n)
Space: O(1)
// Two Sum in Sorted Array function twoSum(arr, target) { let left = 0, right = arr.length - 1; while (left < right) { let sum = arr[left] + arr[right]; if (sum === target) return [left, right]; else if (sum < target) left++; else right--; } return [-1, -1]; }

💡 Key Insight

Start pointers at different positions based on the problem:
• Opposite ends: Sorted array problems
• Same start: Fast-slow problems, sliding window

2 Prefix Sums

📌 What is it?

Prefix sum is like a running total. We pre-calculate cumulative sums so we can quickly find the sum of any subarray. Think of it as "remembering all the previous work"!

Original Array:    [2,  4,  1,  3,  6,  5]
Index:              0   1   2   3   4   5

Building Prefix Sum:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Index 0: prefix[0] = 2           → prefix = [2]
Index 1: prefix[1] = 2 + 4 = 6   → prefix = [2, 6]
Index 2: prefix[2] = 6 + 1 = 7   → prefix = [2, 6, 7]
Index 3: prefix[3] = 7 + 3 = 10  → prefix = [2, 6, 7, 10]
Index 4: prefix[4] = 10 + 6 = 16 → prefix = [2, 6, 7, 10, 16]
Index 5: prefix[5] = 16 + 5 = 21 → prefix = [2, 6, 7, 10, 16, 21]

Result Prefix Array: [2, 6, 7, 10, 16, 21]

Query: Sum from index 2 to 4 (elements [1, 3, 6])
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
prefix[4] - prefix[1] = 16 - 6 = 10
Verify: 1 + 3 + 6 = 10 ✓

🎯 The Magic Formula

Sum of range [i, j] = prefix[j] - prefix[i-1]

If i = 0, just use prefix[j]

Build Time: O(n)
Query Time: O(1)
Space: O(n)
// Build prefix sum array function buildPrefix(arr) { let prefix = new Array(arr.length); prefix[0] = arr[0]; for (let i = 1; i < arr.length; i++) { prefix[i] = prefix[i-1] + arr[i]; } return prefix; } // Get sum of range [left, right] function rangeSum(prefix, left, right) { return left === 0 ? prefix[right] : prefix[right] - prefix[left - 1]; }

💡 Common Applications

  • Range sum queries
  • Subarray sum equals K
  • 2D grid sum queries
  • Equilibrium index finding

3 Sliding Window

📌 What is it?

Imagine a window frame sliding across a picture. Instead of recalculating everything inside the window each time, we add the new element coming in and remove the element going out. It's like moving your gaze across a scene!

Find Maximum Sum Subarray of Size 3:
Array: [2, 1, 5, 1, 3, 2]   Window Size k = 3

Step 1: Window = [2, 1, 5]
        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
        [ 2   1   5 ]  1   3   2
        ←───────→
          Sum = 8 (current max = 8)

Step 2: Slide right - Add 1, Remove 2
        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
          2 [ 1   5   1 ] 3   2
             ←───────→
          Sum = 8 - 2 + 1 = 7 (max still 8)

Step 3: Slide right - Add 3, Remove 1
        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
          2   1 [ 5   1   3 ] 2
                ←───────→
          Sum = 7 - 1 + 3 = 9 (new max = 9) ⭐

Step 4: Slide right - Add 2, Remove 5
        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
          2   1   5 [ 1   3   2 ]
                    ←───────→
          Sum = 9 - 5 + 2 = 6 (max still 9)

✅ Maximum sum = 9 from subarray [5, 1, 3]

🎯 When to Use

  • Fixed window: Max/min sum of size k
  • Variable window: Smallest subarray with sum ≥ K
  • String problems: Longest substring with K distinct chars
  • Problems with contiguous subarrays
Time: O(n)
Space: O(1) or O(k)
// Fixed size sliding window function maxSumSubarray(arr, k) { let windowSum = 0, maxSum = 0; // Build first window for (let i = 0; i < k; i++) { windowSum += arr[i]; } maxSum = windowSum; // Slide the window for (let i = k; i < arr.length; i++) { windowSum = windowSum - arr[i-k] + arr[i]; maxSum = Math.max(maxSum, windowSum); } return maxSum; }

💡 Two Types

Fixed Size: Window size is given (size k)
Dynamic Size: Window grows/shrinks based on condition

4 Kadane's Algorithm

📌 What is it?

Kadane's algorithm finds the maximum sum subarray in O(n) time. The key insight: if a subarray sum becomes negative, discard it and start fresh. Negative prefix can never contribute positively!

Find Maximum Subarray Sum:
Array: [-2, 1, -3, 4, -1, 2, 1, -5, 4]

Walkthrough:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Index  Element   Current Sum    Decision       Max So Far
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  0      -2        -2           Keep (-2)          -2
  1       1     max(-2+1, 1) = 1  Start fresh!      1
  2      -3        1 + (-3) = -2                   1
  3       4     max(-2+4, 4) = 4  Start fresh!      4
  4      -1        4 + (-1) = 3                    4
  5       2        3 + 2 = 5                       5 ⭐
  6       1        5 + 1 = 6                       6 ⭐
  7      -5        6 + (-5) = 1                    6
  8       4        1 + 4 = 5                       6

✅ Maximum subarray sum = 6
   Subarray: [4, -1, 2, 1]

Key: max(current + previous, current)
     = max(extend previous, start fresh)

🎯 The Decision at Each Step

1 Calculate: current + previous_sum
2 If result < current element → START FRESH
3 Else → EXTEND previous subarray
4 Update maximum if current is larger
Time: O(n)
Space: O(1)
function kadane(arr) { let maxSoFar = arr[0]; let maxEndingHere = arr[0]; for (let i = 1; i < arr.length; i++) { // Decision: extend or start fresh? maxEndingHere = Math.max( arr[i], // Start fresh maxEndingHere + arr[i] // Extend ); maxSoFar = Math.max(maxSoFar, maxEndingHere); } return maxSoFar; }

💡 Why It Works

A negative sum prefix can never help maximize the total. If the best subarray ending at position i-1 has negative sum, it's better to start a new subarray at i!

5 Number of Subarrays

📌 What is it?

Counting subarrays that meet certain criteria (sum = K, with certain elements, etc.). The key is using prefix sums + hash maps to count efficiently.

Count Subarrays with Sum = 7:
Array: [3, 4, 7, 2, -3, 1, 4, 2]

Using Prefix Sum + HashMap:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Prefix Sum Flow:
Index:    0    1    2    3    4    5    6    7
Array:    3    4    7    2   -3    1    4    2
Prefix:   3    7   14   16   13   14   18   20

For each position, check: Does (prefix - target) exist?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Index  Prefix   Looking For (prefix - 7)   Found?
  0       3           3 - 7 = -4           No
  1       7           7 - 7 = 0            Yes! (count = 1)
  2      14          14 - 7 = 7            Yes! (count = 2)
  3      16          16 - 7 = 9            No
  4      13          13 - 7 = 6            No
  5      14          14 - 7 = 7            Yes! (count = 3)
  6      18          18 - 7 = 11           No
  7      20          20 - 7 = 13           Yes! (count = 4)

✅ Total subarrays with sum 7 = 4

HashMap state at end: {0:1, 3:1, 7:2, 14:2, 16:1, 13:1, 18:1}

🎯 The Pattern

Core Formula: count += map.get(prefixSum - target)

At each position, we ask: "How many times have we seen a prefix sum that equals (current - target)?"

Time: O(n)
Space: O(n)
function subarraySum(arr, k) { let count = 0, prefixSum = 0; let map = new Map([[0, 1]]); // prefix 0 occurs once for (let num of arr) { prefixSum += num; // Check if (prefixSum - k) exists if (map.has(prefixSum - k)) { count += map.get(prefixSum - k); } // Store current prefix sum map.set(prefixSum, (map.get(prefixSum) || 0) + 1); } return count; }

💡 Total Number of Subarrays

For array of length n: n × (n+1) / 2 subarrays total

Example: n = 5 → 5 × 6 / 2 = 15 subarrays

6 Fast & Slow Pointers

📌 What is it?

Two pointers moving at different speeds through a sequence. Like a race where one runner is twice as fast as another. If there's a cycle, they'll eventually meet!

Detecting a Cycle in Linked List:

List: 1 → 2 → 3 → 4 → 5 → (back to 3, cycle!)

Position tracking:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 0:  Slow at 1, Fast at 1
         [1] → [2] → [3] → [4] → [5]
          ↑
         S,F

Step 1:  Slow moves 1, Fast moves 2
         [1] → [2] → [3] → [4] → [5]
                ↑      ↑
                S      F

Step 2:  Slow at 3, Fast at 5
         [1] → [2] → [3] → [4] → [5]
                       ↑             ↑
                       S             F

Step 3:  Slow at 4, Fast back at 3 (cycle!)
         [1] → [2] → [3] → [4] → [5]
                 ↑      ↑
                 F      S

Step 4:  Slow at 5, Fast at 5
         [1] → [2] → [3] → [4] → [5]
                                ↑↑
                                SF
         
         ✅ CYCLE DETECTED! They met at node 5!

🎯 Why It Works

Think of it like a circular track. If fast runs 2 laps for every 1 lap of slow, eventually fast will "lap" slow and they'll meet. Distance between them increases by 1 each step, and since the track is finite, they must meet.

Time: O(n)
Space: O(1)
// Cycle detection function hasCycle(head) { let slow = head, fast = head; while (fast && fast.next) { slow = slow.next; // Move 1 step fast = fast.next.next; // Move 2 steps if (slow === fast) return true; // Cycle! } return false; } // Find middle of linked list function findMiddle(head) { let slow = head, fast = head; while (fast && fast.next) { slow = slow.next; fast = fast.next.next; } return slow; // When fast reaches end, slow is at middle }

💡 Applications

  • Cycle detection in linked list
  • Find middle of linked list
  • Happy number problem
  • Cycle start point detection

7 Depth-First Search (DFS)

📌 What is it?

DFS explores as deep as possible before backtracking. Like exploring a maze by always taking the first path you see, and only going back when you hit a dead end. Uses a stack (or recursion)!

DFS Traversal of a Tree:

            Tree:           Visited Order:
            
              A              1. Start at A
             /|\             2. Go to B (first child)
            / | \            3. Go to D (B's first child)
           B  C  D           4. Backtrack, go to E
          /|    |            5. Backtrack to A
         E F    G            6. Go to C
                             7. Backtrack to A
        Order: A → B → E → F → C → D → G

DFS on Grid (like finding path in maze):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    0   1   2   3   4
  ┌───┬───┬───┬───┬───┐
0 │ S │   │ █ │   │   │   S = Start
  ├───┼───┼───┼───┼───┤   E = End
1 │ █ │   │ █ │   │   │   █ = Wall
  ├───┼───┼───┼───┼───┤   • = Visited
2 │   │   │   │   │ █ │   → = Path
  ├───┼───┼───┼───┼───┤
3 │   │ █ │ █ │   │ E │
  └───┴───┴───┴───┴───┘

DFS explores each direction completely before trying next:
  1. Try RIGHT as far as possible
  2. Hit wall? BACKTRACK
  3. Try DOWN, then RIGHT again
  4. Continue until found or all explored

🎯 DFS Template

1 Mark current node as visited
2 Process current node (add to result, check condition)
3 For each unvisited neighbor, recurse DFS
4 Backtrack if needed (for path problems)
Time: O(V + E)
Space: O(V)
// DFS for Graph (recursive) function dfs(graph, node, visited) { visited.add(node); console.log(node); // Process node for (let neighbor of graph[node]) { if (!visited.has(neighbor)) { dfs(graph, neighbor, visited); } } } // DFS for Grid function dfsGrid(grid, row, col, visited) { const ROWS = grid.length, COLS = grid[0].length; const dirs = [[1,0], [-1,0], [0,1], [0,-1]]; if (row < 0 || row >= ROWS || col < 0 || col >= COLS) return; if (visited.has(`${row},${col}`)) return; visited.add(`${row},${col}`); // Process cell here for (let [dr, dc] of dirs) { dfsGrid(grid, row+dr, col+dc, visited); } }

💡 When to Use DFS

  • Find all paths between nodes
  • Check if path exists
  • Topological sort
  • Detect cycles
  • Solve mazes and puzzles

8 Breadth-First Search (BFS)

📌 What is it?

BFS explores level by level, like ripples spreading from a stone thrown in water. Uses a queue to process nodes in order they were discovered. Perfect for finding shortest paths!

BFS Traversal of a Tree:

            Tree:           Level Order:
            
              A              Level 0: A
             /|\             Level 1: B, C, D
            / | \            Level 2: E, F, G
           B  C  D           
          /|    |            
         E F    G            

Queue progression:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Step 1: Queue = [A]           → Process A
Step 2: Queue = [B,C,D]       → Process B
Step 3: Queue = [C,D,E,F]     → Process C
Step 4: Queue = [D,E,F]       → Process D
Step 5: Queue = [E,F,G]       → Process E, F, G

BFS on Grid (shortest path):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    0   1   2   3   4
  ┌───┬───┬───┬───┬───┐
0 │ S │ 1 │ █ │   │   │   Numbers show distance
  ├───┼───┼───┼───┼───┤   from start
1 │ █ │ 2 │ █ │   │   │
  ├───┼───┼───┼───┼───┤
2 │ 3 │ 3 │ 4 │ 5 │ █ │
  ├───┼───┼───┼───┼───┤
3 │ 4 │ █ │ █ │ 6 │ E │
  └───┴───┴───┴───┴───┘

BFS finds shortest path: S → (0,1) → (1,1) → (2,0) → (2,1) → (2,2) → (2,3) → (3,3) → E
Distance = 8 steps

🎯 BFS Template

1 Add start node to queue, mark visited
2 While queue not empty: dequeue front
3 Process current node
4 Add all unvisited neighbors to queue
Time: O(V + E)
Space: O(V)
// BFS for Graph function bfs(graph, start) { let visited = new Set(); let queue = [start]; visited.add(start); while (queue.length > 0) { let node = queue.shift(); // Dequeue front console.log(node); // Process for (let neighbor of graph[node]) { if (!visited.has(neighbor)) { visited.add(neighbor); queue.push(neighbor); // Enqueue } } } } // BFS for Grid (shortest path) function bfsGrid(grid, start, end) { const ROWS = grid.length, COLS = grid[0].length; const dirs = [[1,0], [-1,0], [0,1], [0,-1]]; const queue = [[...start, 0]]; // [row, col, distance] const visited = new Set(); while (queue.length) { const [r, c, dist] = queue.shift(); if (r === end[0] && c === end[1]) return dist; for (let [dr, dc] of dirs) { const nr = r + dr, nc = c + dc; const key = `${nr},${nc}`; if (nr >= 0 && nr < ROWS && nc >= 0 && nc < COLS && !visited.has(key) && grid[nr][nc] !== '█') { visited.add(key); queue.push([nr, nc, dist + 1]); } } } return -1; // Not reachable }

💡 When to Use BFS

  • Shortest path in unweighted graph
  • Level order traversal
  • Find nodes at distance k
  • Check if graph is bipartite

10 Adjacency List BFS/DFS

📌 What is it?

Adjacency list stores graph as a hash map of lists: each node maps to its neighbors. More memory efficient than adjacency matrix for sparse graphs.

Graph Representation:

    Graph:                  
         A ───── B
        /│      │\
       / │      │ \
      C  │      │  D
       \ │      │ /
        \|      │/

13 Topological Sort

📌 What is it?

Ordering of vertices in a DAG (Directed Acyclic Graph) where for every edge A→B, A comes before B. Think of it as course prerequisites: you must take CS101 before CS201!

Course Schedule Problem:

Prerequisites: [[1,0], [2,0], [3,1], [3,2]]
              (to take 1, need 0 first)

Graph:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
        0 ──────┬──────→ 1
        │       │        │
        │       │        │
        ↓       ↓        ↓
        2 ───────→ 3 ←───┘

Adjacency List:
{
    0: [1, 2],   ← Course 0 unlocks 1 and 2
    1: [3],      ← Course 1 unlocks 3
    2: [3],      ← Course 2 unlocks 3
    3: []        ← No courses depend on 3
}

In-degree: {0: 0, 1: 1, 2: 1, 3: 2}
           ↑              ↑
        No deps        Needs 1 AND 2

Kahn's Algorithm (BFS):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: Queue = [0]  (in-degree = 0)
        ┌───┐
        │ 0 │ → in-degree: {1:0, 2:0, 3:2}
        └───┘

Step 2: Process 0, reduce neighbors' in-degrees
        Queue = [1, 2]
        ┌───┬───┐
        │ 1 │ 2 │ → in-degree: {3:1, 3:1}
        └───┴───┘
        Result: [0]

Step 3: Process 1
        Queue = [2]
        ┌───┐
        │ 2 │ → in-degree: {3:0}
        └───┘
        Result: [0, 1]

Step 4: Process 2
        Queue = [3]
        ┌───┐
        │ 3 │ → in-degree: {}
        └───┘
        Result: [0, 1, 2]

Step 5: Process 3
        Queue = []
        Result: [0, 1, 2, 3]

✅ Valid Order: [0, 1, 2, 3] or [0, 2, 1, 3]

🎯 Kahn's Algorithm Steps

1 Calculate in-degree for all nodes
2 Add all nodes with in-degree = 0 to queue
3 Dequeue, add to result, reduce neighbors' in-degrees
4 If neighbor's in-degree becomes 0, enqueue it
5 If result has all nodes → valid, else → cycle exists

⚠️ Important

Topological sort only works on DAGs. If there's a cycle, we can't produce a valid ordering. Detect this when result.length < number of vertices.

📌 What is it?

A stack where elements are always sorted (increasing or decreasing). When a new element "breaks" the order, we pop elements until order is restored. Perfect for finding next greater/smaller element!

// Merge Intervals function merge(intervals) { if (!intervals.length) return []; // Sort by start time intervals.sort((a, b) => a[0] - b[0]); const result = [intervals[0]]; for (let i = 1; i < intervals.length; i++) { const [start, end] = intervals[i]; const lastEnd = result[result.length - 1][1]; if (start <= lastEnd) { // Overlapping - merge by extending end result[result.length - 1][1] = Math.max(lastEnd, end); } else { // Non-overlapping - add new interval result.push([start, end]); } } return result; } // Insert Interval function insert(intervals, newInterval) { const result = []; let i = 0; // Add all intervals before newInterval while (i < intervals.length && intervals[i][1] < newInterval[0]) { result.push(intervals[i]); i++; } // Merge overlapping intervals while (i < intervals.length && intervals[i][0] <= newInterval[1]) { newInterval[0] = Math.min(newInterval[0], intervals[i][0]); newInterval[1] = Math.max(newInterval[1], intervals[i][1]); i++; } result.push(newInterval); // Add remaining intervals while (i < intervals.length) { result.push(intervals[i]); i++; } return result; } // Non-overlapping Intervals (min removals) function eraseOverlapIntervals(intervals) { intervals.sort((a, b) => a[1] - b[1]); // Sort by end let count = 0; let prevEnd = intervals[0][1]; for (let i = 1; i < intervals.length; i++) { if (intervals[i][0] < prevEnd) { count++; // Remove this interval } else { prevEnd = intervals[i][1]; } } return count; }
