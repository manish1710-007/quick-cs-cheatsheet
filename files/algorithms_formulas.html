Asymptotic Notations

Big-O (Upper Bound)

f(n) = O(g(n))

∃c > 0, n₀ > 0: f(n) ≤ c·g(n) ∀n ≥ n₀

"Worst case - grows no faster than"

Big-Ω (Lower Bound)

f(n) = Ω(g(n))

∃c > 0, n₀ > 0: f(n) ≥ c·g(n) ∀n ≥ n₀

"Best case - grows at least as fast as"

Big-Θ (Tight Bound)

f(n) = Θ(g(n))

f(n) = O(g(n)) AND f(n) = Ω(g(n))

"Average case - grows exactly as fast as"

Little-o (Strict Upper)

f(n) = o(g(n))

lim(n→∞) f(n)/g(n) = 0

"Grows strictly slower than"

Little-ω (Strict Lower)

f(n) = ω(g(n))

lim(n→∞) f(n)/g(n) = ∞

"Grows strictly faster than"

Master Theorem

General Form

T(n) = aT(n/b) + f(n)

Where: a ≥ 1 (subproblems), b > 1 (size reduction), f(n) = work to divide/combine

Case 1: f(n) = O(n^(log_b a - ε))

T(n) = Θ(n^(log_b a))

Work at leaves dominates

Example:

T(n) = 2T(n/2) + n⁰ = 2T(n/2) + 1

a=2, b=2, log₂2 = 1, f(n)=1=O(n^0.5)

T(n) = Θ(n)

Case 2: f(n) = Θ(n^(log_b a) · logᵏ n)

T(n) = Θ(n^(log_b a) · log^(k+1) n)

Work is evenly distributed

Example:

T(n) = 2T(n/2) + n

a=2, b=2, log₂2 = 1, f(n)=n=Θ(n¹log⁰n)

T(n) = Θ(n log n)

Case 3: f(n) = Ω(n^(log_b a + ε))

T(n) = Θ(f(n))

Work at root dominates (if regularity condition holds)

Example:

T(n) = 2T(n/2) + n²

a=2, b=2, log₂2 = 1, f(n)=n²=Ω(n^1.5)

T(n) = Θ(n²)

Sorting Algorithm Details

Bubble Sort

for i = 0 to n-1:
  for j = 0 to n-i-1:
    if arr[j] > arr[j+1]:
      swap(arr[j], arr[j+1])

Inversions: Maximum n(n-1)/2

Swaps: Equal to number of inversions

Selection Sort

for i = 0 to n-1:
  min_idx = i
  for j = i+1 to n:
    if arr[j] < arr[min_idx]:
      min_idx = j
  swap(arr[i], arr[min_idx])

Swaps: Always exactly n-1

Insertion Sort

for i = 1 to n-1:
  key = arr[i]
  j = i - 1
  while j >= 0 and arr[j] > key:
    arr[j+1] = arr[j]
    j = j - 1
  arr[j+1] = key

Best for: Nearly sorted arrays, small datasets

Merge Sort

MergeSort(arr, l, r):
  if l < r:
    m = (l + r) / 2
    MergeSort(arr, l, m)
    MergeSort(arr, m+1, r)
    Merge(arr, l, m, r)

Stable: Yes, guaranteed O(n log n)

Quick Sort

QuickSort(arr, low, high):
  if low < high:
    pi = Partition(arr, low, high)
    QuickSort(arr, low, pi-1)
    QuickSort(arr, pi+1, high)

Pivot strategies: First, Last, Random, Median-of-3

Heap Sort

BuildMaxHeap(arr) // O(n)
for i = n-1 downto 1:
  swap(arr[0], arr[i])
  Heapify(arr, 0, i) // O(log n)

In-place: Yes, not stable

Lower Bound for Comparison Sorting

Decision Tree Analysis

Minimum comparisons = ⌈log₂(n!)⌉ ≈ n log n - 1.44n

Using Stirling's approximation: log₂(n!) = Θ(n log n)

Conclusion: No comparison-based sort can do better than Ω(n log n)

Binary Search Variants

Standard Binary Search

while low <= high:
  mid = low + (high - low) / 2
  if arr[mid] == target: return mid
  elif arr[mid] < target: low = mid + 1
  else: high = mid - 1

Note: Use low + (high-low)/2 to avoid overflow

Lower Bound (First ≥ target)

while low < high:
  mid = low + (high - low) / 2
  if arr[mid] >= target: high = mid
  else: low = mid + 1
return low

Finds first position where element ≥ target

Upper Bound (First > target)

while low < high:
  mid = low + (high - low) / 2
  if arr[mid] > target: high = mid
  else: low = mid + 1
return low

Finds first position where element > target

Find Peak Element

while low < high:
  mid = low + (high - low) / 2
  if arr[mid] > arr[mid+1]: high = mid
  else: low = mid + 1
return low

Finds a peak (not necessarily global max)

Dynamic Programming

DP Fundamentals

When to Use DP

  • Optimal Substructure: Optimal solution contains optimal solutions to subproblems
  • Overlapping Subproblems: Same subproblems solved multiple times

DP vs Other Techniques

TechniqueSubproblems
Divide & ConquerNon-overlapping
Dynamic ProgrammingOverlapping
GreedyNo backtracking

Classic DP Problems

Fibonacci Numbers

F(n) = F(n-1) + F(n-2)
// Bottom-up DP
dp[0] = 0, dp[1] = 1
for i = 2 to n:
  dp[i] = dp[i-1] + dp[i-2]

Space optimization: O(n) → O(1)

Longest Common Subsequence (LCS)

If X[i] == Y[j]: dp[i][j] = dp[i-1][j-1] + 1
Else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])

Time: O(mn), Space: O(mn)

Space optimization: O(min(m,n))

0/1 Knapsack Problem

dp[i][w] = max(dp[i-1][w], dp[i-1][w-wᵢ] + vᵢ)

where wᵢ ≤ w (item i can fit)

Time: O(nW), Space: O(nW) → O(W)

Example:

n=3 items, W=50

Weights: [10, 20, 30], Values: [60, 100, 120]

Optimal: Items 1,2 → Value = 160

Unbounded Knapsack

dp[w] = max(dp[w], dp[w-wᵢ] + vᵢ)

Items can be used multiple times

Time: O(nW), Space: O(W)

Longest Increasing Subsequence (LIS)

dp[i] = max(dp[j] + 1) for all j < i where arr[j] < arr[i]

DP Time: O(n²)

Binary Search: O(n log n)

Edit Distance

If s1[i] == s2[j]: dp[i][j] = dp[i-1][j-1]
Else: dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])

Operations: Insert, Delete, Replace

Time: O(mn)

Matrix Chain Multiplication

dp[i][j] = min(dp[i][k] + dp[k+1][j] + p[i-1]·p[k]·p[j])

for k from i to j-1

Time: O(n³), Space: O(n²)

Coin Change

dp[amount] = min(dp[amount], dp[amount - coin] + 1)

Min coins: O(amount × n_coins)

dp[amount] += dp[amount - coin]

Number of ways: O(amount × n_coins)

Subset Sum

dp[i][j] = dp[i-1][j] OR dp[i-1][j-arr[i]]

Check if subset sums to target

Time: O(n × sum)

Rod Cutting

dp[n] = max(price[i] + dp[n-i-1]) for i = 0 to n-1

Maximize profit from cutting rod

Time: O(n²)

DP on Strings

Longest Palindromic Subsequence

If s[i] == s[j]: dp[i][j] = 2 + dp[i+1][j-1]
Else: dp[i][j] = max(dp[i+1][j], dp[i][j-1])

Time: O(n²)

Longest Common Substring

If X[i] == Y[j]: dp[i][j] = dp[i-1][j-1] + 1
Else: dp[i][j] = 0

Track maximum dp[i][j]

Time: O(mn)

Regular Expression Matching

dp[i][j] = dp[i-1][j-1] if match
If p[j] == '*': dp[i][j] = dp[i][j-2] OR dp[i-1][j]

Time: O(mn)

DP on Trees

Maximum Independent Set on Tree

dp[node][0] = sum of max(dp[child][0], dp[child][1])
dp[node][1] = node.value + sum(dp[child][0])

dp[node][0] = node NOT included

dp[node][1] = node included

Greedy Algorithms

Greedy Properties

When Greedy Works

  • Greedy Choice Property: Local optimal leads to global optimal
  • Optimal Substructure: Solution contains optimal solutions to subproblems

Greedy vs DP

AspectGreedyDP
ChoiceMake onceConsider all
SpeedFasterSlower
CorrectnessNot alwaysAlways

Classic Greedy Problems

Activity Selection

Goal: Max number of non-overlapping activities

Strategy: Sort by finish time, greedily select

Sort activities by finish time
selected = [first activity]
last_finish = finish[0]
for i = 1 to n:
  if start[i] >= last_finish:
    selected.add(i)
    last_finish = finish[i]

Time: O(n log n) for sorting

Fractional Knapsack

Goal: Maximize value with weight limit

Strategy: Sort by value/weight ratio, take highest first

Sort items by (value/weight) descending
remaining = W, total_value = 0
for each item:
  take = min(item.weight, remaining)
  total_value += take * item.ratio
  remaining -= take

Time: O(n log n)

Huffman Coding

Goal: Optimal prefix-free encoding

Strategy: Build tree from bottom, combine lowest freq

Total bits = Σ(freqᵢ × depthᵢ)

Time: O(n log n) with priority queue

Job Sequencing with Deadlines

Goal: Maximize profit, each job takes 1 unit

Strategy: Sort by profit, place as late as possible

Max jobs = min(n, max_deadline)

Time: O(n²) or O(n log n) with DSU

Minimum Spanning Tree

Kruskal's: Sort edges, add if no cycle (Union-Find)

Prim's: Start from vertex, add minimum edge to tree

Time (Kruskal): O(E log E)

Time (Prim): O(E log V) with heap

Dijkstra's Shortest Path

Goal: Shortest path from source (non-negative weights)

Strategy: Greedily pick minimum distance vertex

Time: O((V + E) log V) with heap

Important

Does NOT work with negative edges!

Greedy Exchange Argument

Proving Greedy Correctness

Exchange Argument: Show that any optimal solution can be transformed into the greedy solution without making it worse.

Steps:

  1. Assume an optimal solution different from greedy
  2. Find the first point where optimal differs from greedy
  3. Show that swapping to greedy choice doesn't hurt
  4. Conclude greedy is optimal

Graph Traversals

Breadth-First Search (BFS)

BFS(start):
  queue = [start], visited = {start}
  while queue:
    u = queue.pop_front()
    for v in adj[u]:
      if v not in visited:
        visited.add(v)
        queue.push_back(v)

Time: O(V + E), Space: O(V)

Uses: Shortest path (unweighted), level order

Depth-First Search (DFS)

DFS(u):
  visited.add(u)
  for v in adj[u]:
    if v not in visited:
      DFS(v)

Time: O(V + E), Space: O(V)

Uses: Cycle detection, topological sort, SCC

Dijkstra's Algorithm

Dijkstra(source):
  dist[source] = 0, others = ∞
  pq = priority_queue({0, source})
  while pq:
    d, u = pq.pop()
    if d > dist[u]: continue
    for v, w in adj[u]:
      if dist[u] + w < dist[v]:
        dist[v] = dist[u] + w
        pq.push({dist[v], v})

Bellman-Ford Algorithm

BellmanFord(source):
  dist[source] = 0, others = ∞
  for i = 1 to V-1:
    for each edge (u, v, w):
      if dist[u] + w < dist[v]:
        dist[v] = dist[u] + w
  // Check for negative cycle
  for each edge (u, v, w):
    if dist[u] + w < dist[v]:
      return "Negative cycle"

Floyd-Warshall Algorithm

// Initialize dist[i][j] = weight(i,j) or ∞
for k = 1 to n:
  for i = 1 to n:
    for j = 1 to n:
      dist[i][j] = min(dist[i][j],
                  dist[i][k] + dist[k][j])

All-pairs shortest paths

Minimum Spanning Tree (MST)

Kruskal's Algorithm

Sort edges by weight
for each edge (u, v, w) in sorted:
  if find(u) != find(v):
    union(u, v)
    add edge to MST

Time: O(E log E) or O(E α(V))

Uses Union-Find (Disjoint Set Union)

Prim's Algorithm

Prim(start):
  inMST = set(), pq = [{0, start}]
  while pq:
    w, u = pq.pop()
    if u in inMST: continue
    inMST.add(u)
    for v, w in adj[u]:
      if v not in inMST:
        pq.push({w, v})

Time: O(E log V)

MST Properties

MST edges = V - 1
MST is unique if all edge weights are distinct

Cut Property: Minimum weight edge crossing a cut is in MST

Cycle Property: Maximum weight edge in a cycle is NOT in MST

Topological Sort

Kahn's Algorithm (BFS)

Calculate in-degree of all vertices
queue = vertices with in-degree 0
while queue:
  u = queue.pop()
  result.add(u)
  for v in adj[u]:
    in_degree[v] -= 1
    if in_degree[v] == 0:
      queue.push(v)

Time: O(V + E)

If result has < V vertices → cycle exists

DFS-based Topological Sort

DFS(u):
  visited.add(u)
  for v in adj[u]:
    if v not in visited:
      DFS(v)
  stack.push(u) // After visiting all neighbors
// Pop all from stack for result

Time: O(V + E)

Strongly Connected Components (SCC)

Kosaraju's Algorithm

// Step 1: DFS on original graph
Fill stack with finish order
// Step 2: Transpose graph
// Step 3: DFS on transpose in stack order
while stack:
  u = stack.pop()
  if u not in visited:
    DFS(u) // One SCC

Time: O(V + E)

DFS(u):
  disc[u] = low[u] = time++
  stack.push(u)

    if v not in visited:
      DFS(v)
      low[u] = min(low[u], low[v])
    elif v in stack:
      low[u] = min(low[u], disc[v])
  if low[u] == disc[u]:
    // Pop stack until u - one SCC

Time: O(V + E)

Other Graph Algorithms

Cycle Detection

Undirected: DFS with parent tracking

Directed: DFS with recursion stack or topological sort

Has cycle ⟺ back edge exists in DFS

Union-Find (Disjoint Set)

find(x): // Path compression
  if parent[x] != x:
    parent[x] = find(parent[x])
  return parent[x]

union(x, y): // Union by rank
  px, py = find(x), find(y)
  if rank[px] < rank[py]: parent[px] = py
  else if rank[px] > rank[py]: parent[py] = px
  else: parent[py] = px, rank[px]++

Time: O(α(n)) per operation (inverse Ackermann)

Bipartite Check

Graph is bipartite ⟺ can color with 2 colors

Use BFS/DFS with alternating colors

Bipartite ⟺ No odd-length cycle

Eulerian Path/Circuit

Eulerian Circuit: All vertices have even degree

Eulerian Path: Exactly 0 or 2 vertices have odd degree

Algorithm: Hierholzer's O(V + E)

Bridge (u,v) if low[v] > disc[u]

Branch and Bound

Concept

Branch: Divide problem into subproblems

Bound: Calculate lower/upper bound for each subproblem

Prune: Discard subproblems that can't lead to optimal solution

Traveling Salesman Problem (TSP)

Use reduced cost matrix for lower bound

Prune branches where lower bound > current best

Time: Still exponential, but much faster than brute force

String Algorithms

Pattern Matching

Time: O(m × n)

KMP Algorithm

Time: O(m + n)

Uses failure function (LPS array)

LPS[i] = longest proper prefix of pat[0..i] that is also suffix

// Build LPS array: O(m)
// Search: O(n)

Rabin-Karp Algorithm

Average: O(m + n), Worst: O(m × n)

Uses rolling hash function

hash = (d × hash + char) mod q
Rolling: hash = (d(hash - old × h) + new) mod q

h = d^(m-1) mod q

Boyer-Moore Algorithm

Z-Algorithm

Time: O(m + n)

Aho-Corasick (Multiple Patterns)

Uses trie with failure links (like KMP on trie)

Brian Kernighan: count while n > 0, n &= (n-1)

Time: O(number of set bits)

Population count: Built-in popcount()

Subset Generation

All subsets: O(2ⁿ × n)

Each number 0 to 2ⁿ-1 represents a subset

Bit i set → element i included

Power of 2 Check

n > 0 AND (n & (n-1)) == 0

True if n is power of 2

Mathematical Algorithms

GCD (Euclidean Algorithm)

gcd(a, b) = gcd(b, a mod b)
gcd(a, b):
  while b != 0:
    a, b = b, a % b
  return a

Time: O(log min(a, b))

Extended Euclidean Algorithm

ax + by = gcd(a, b)

Finds x, y such that above holds

Used for modular inverse

Modular Exponentiation

a^b mod m
power(a, b, m):
  result = 1
  while b > 0:
    if b & 1: result = result * a % m
    a = a * a % m
    b >>= 1
  return result

Time: O(log b)

Prime Testing (Miller-Rabin)

Time: O(k × log³n)

k = number of iterations (witnesses)

Probabilistic: Can give false positives

Sieve of Eratosthenes

Time: O(n log log n)

Find all primes up to n

Space: O(n)

Segmented Sieve

Time: O(n log log n)

Find primes in range [L, R]

Space: O(√R + (R-L))

Segment Trees & Fenwick Trees

Segment Tree

Build: O(n), Query: O(log n), Update: O(log n)

Range queries: sum, min, max, gcd, etc.

Space: O(4n)

Tree stored in array: children of i at 2i, 2i+1

Lazy Propagation

Deferred updates for range updates

Range Update + Range Query: O(log n)

Store pending updates in lazy array

Fenwick Tree (Binary Indexed Tree)

Build: O(n log n), Query: O(log n), Update: O(log n)

Prefix sum queries, point updates

Space: O(n)

i & (-i) = lowest set bit (LSB)

Parent: i + LSB (update), i - LSB (query)

NP Completeness

Complexity Classes

P: Problems solvable in polynomial time

NP: Problems verifiable in polynomial time

NP-Hard: At least as hard as all NP problems

NP-Complete: NP ∩ NP-Hard

NP-Complete Problems

  • SAT (Boolean Satisfiability)
  • 3-SAT
  • Vertex Cover
  • Hamiltonian Path/Cycle
  • Traveling Salesman (decision)
  • Subset Sum
  • Graph Coloring
  • Clique

Reductions

To prove NP-Complete:

  1. Show problem is in NP
  2. Reduce known NP-Complete problem to it

Reduction must be polynomial time
