Directory Purposes

DirectoryPurposeContents
/binEssential user binariesls, cp, mv, cat, bash
/sbinSystem binariesinit, reboot, fdisk, iptables
/etcSystem configurationpasswd, hosts, fstab, ssh/
/homeUser home directories/home/username
/varVariable datalogs, mail, spool, www
/usrUser programsbin/, lib/, share/, local/
/tmpTemporary filesCleared on reboot
/procProcess informationVirtual filesystem (kernel)
/devDevice filesnull, zero, random, sda, tty
/bootBoot loader filesvmlinuz, initrd, grub/
💡 Key Differences from Windows:
  • No "C:" drives - everything starts from / (root)
  • Paths use forward slash (/) not backslash (\)
  • Case-sensitive: File.txt ≠ file.txt
  • Hidden files start with dot (.config, .bashrc)
  • No file extensions required for execution (permissions matter)

🔍 Navigation & File Operations

Directory Navigation

# Print working directory pwd # Change directory cd /home/user # Absolute path cd Documents # Relative path cd .. # Go up one level cd ../.. # Go up two levels cd ~ # Go to home directory cd - # Go to previous directory

Listing Files

ls # List files ls -l # Long format (permissions, owner, size) ls -a # All files (including hidden) ls -la # Combined: long + all ls -lh # Human-readable sizes ls -lt # Sort by modification time ls -lS # Sort by size ls -R # Recursive listing ls -d */ # List only directories ls -i # Show inode numbers

Creating & Removing

# Create directories mkdir dirname mkdir -p a/b/c/d # Create parent directories mkdir {dir1,dir2,dir3} # Create multiple # Create empty file / update timestamp touch file.txt touch {a,b,c}.txt # Create a.txt, b.txt, c.txt # Remove files rm file.txt rm -i file.txt # Interactive (confirm) rm -f file.txt # Force (no prompt) # Remove directories rmdir emptydir # Only empty directories rm -r dirname # Recursive (directory + contents) rm -rf dirname # Force recursive (DANGEROUS!)
🚨 DANGER: rm -rf / or rm -rf * will delete everything. There's no trash bin in Linux!

🔐 File Permissions

Permission Structure

Permission String: -rwxr-xr-- Position: 1 234 567 890 │ │ │ └── Others (r--) │ │ └────── Group (r-x) │ └────────── Owner (rwx) └──────────── Type (- = file, d = directory, l = link) Permission Values: r (read) = 4 w (write) = 2 x (execute) = 1 - (none) = 0 Common Combinations: rwx = 4+2+1 = 7 (read, write, execute) rw- = 4+2+0 = 6 (read, write) r-x = 4+0+1 = 5 (read, execute) r-- = 4+0+0 = 4 (read only) Common Settings: 755 = rwxr-xr-x (directories, executables) 644 = rw-r--r-- (regular files) 600 = rw------- (private files like keys) 700 = rwx------ (private directories)
rwx (Owner)
r-x (Group)
r-- (Others)

chmod - Change Permissions

# Numeric mode chmod 755 script.sh # rwxr-xr-x chmod 644 file.txt # rw-r--r-- chmod 600 id_rsa # rw------- chmod 700 ~/.ssh # rwx------ # Symbolic mode chmod +x script.sh # Add execute for all chmod u+x file # Add execute for owner chmod g-w file # Remove write for group chmod o=r file # Set others to read only chmod a+r file # Add read for all chmod u=rwx,g=rx,o=r file # Set all at once # Recursive chmod -R 755 directory/ # Reference file chmod --reference=reffile target

chown - Change Ownership

# Change owner chown user file.txt # Change owner and group chown user:group file.txt # Change group only chown :group file.txt chgrp group file.txt # Recursive chown -R user:group directory/

📜 Shell Scripting Basics

Script Structure

#!/bin/bash # Shebang - interpreter # Variables NAME="John" AGE=25 FILES=$(ls) # Command substitution # Use variables echo "Hello, $NAME" echo "Age: ${AGE}" # Special variables $0 # Script name $1-$9 # Positional arguments $# # Number of arguments $@ # All arguments as array $* # All arguments as string $? # Exit code of last command $$ # Process ID $! # PID of last background job # Run script chmod +x script.sh ./script.sh arg1 arg2

Conditionals

if [ $AGE -gt 18 ]; then echo "Adult" elif [ $AGE -eq 18 ]; then echo "Just turned adult" else echo "Minor" fi # File tests [ -f file ] # Exists and is file [ -d dir ] # Exists and is directory [ -e path ] # Exists [ -r file ] # Readable [ -w file ] # Writable [ -x file ] # Executable [ -s file ] # Non-empty # String tests [ -z "$str" ] # Empty string [ -n "$str" ] # Non-empty string [ "$a" = "$b" ] # Equal

Loops

# For loop for i in {1..5}; do echo "Number: $i" done for file in *.txt; do echo "Processing $file" done # While loop count=0 while [ $count -lt 5 ]; do echo $count ((count++)) done # Until loop until [ $count -ge 10 ]; do echo $count ((count++)) done # Loop control break # Exit loop continue # Next iteration

Functions

# Define function greet() { local name=$1 echo "Hello, $name!" } # Call function greet "Alice" # Function with return add() { local sum=$(($1 + $2)) echo $sum } result=$(add 5 3) # Exit codes check_file() { [ -f "$1" ] && return 0 return 1 }

Practical Examples

#!/bin/bash # Backup script BACKUP_DIR="/backup" DATE=$(date +%Y%m%d) for dir in "$@"; do if [ -d "$dir" ]; then basename=$(basename "$dir") tar -czf "${BACKUP_DIR}/${basename}_${DATE}.tar.gz" "$dir" echo "Backed up $basename" else echo "Error: $dir not found" >&2 fi done

✅ Linux Best Practices

🔒 Security

  • Use sudo, not root login
  • Keep system updated regularly
  • Use SSH keys, disable password auth
  • Configure firewall (ufw/iptables)
  • Set proper file permissions
  • Review logs regularly
  • Use fail2ban for brute force protection

⚙️ System Administration

  • Document all changes made
  • Test changes in staging first
  • Use version control for configs
  • Set up monitoring and alerts
  • Automate with scripts/Ansible
  • Regular backups with tested restore
  • Keep scripts in version control

📝 Scripting

  • Always use shebang line
  • Quote variables: "$var"
  • Use set -e for error exit
  • Add meaningful comments
  • Validate inputs
  • Use functions for reusability
  • Log script output

🗂️ File Management

  • Follow FHS directory structure
  • Use meaningful file names
  • Clean up temp files
  • Use .gitignore properly
  • Compress old logs
  • Set appropriate permissions
  • Use symlinks for flexibility
