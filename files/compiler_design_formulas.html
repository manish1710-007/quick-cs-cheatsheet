Finite Automata for Lexical Analysis

NFA to DFA Conversion

DFA states = Subsets of NFA states
Max DFA states = 2^n (n = NFA states)
Subset construction algorithm converts NFA to equivalent DFA.

DFA Minimization

Min states = Number of equivalence classes
Partition refinement algorithm groups indistinguishable states.

Buffering Techniques

Double Buffer

Buffer size: N characters each
Two alternating buffers. While one is processed, other is loaded. Uses sentinels to detect end.

Buffer Pairs

Sentinel character at end of each buffer

🌳 Syntax Analysis (Parsing)

Context-Free Grammar

Formal Definition
G = (V, T, P, S)
Where:
• V = Set of non-terminals (variables)
• T = Set of terminals (tokens)
• P = Set of productions (rules)
• S = Start symbol
📌 Example: Grammar for Expressions
E → E + T | T
T → T * F | F
F → ( E ) | id

Types of Parsing

Top-Down Parsing

• Starts from start symbol
• Constructs parse tree from root to leaves
• Uses leftmost derivation
• Types: Recursive descent, LL(1)
⚠️ Problem:
Left recursion must be eliminated for top-down parsers.

Bottom-Up Parsing

• Starts from input string
• Constructs parse tree from leaves to root
• Uses reverse of rightmost derivation
• Types: Operator precedence, LR parsers
Reduce: Replace RHS with LHS
Shift: Move input to stack

Left Recursion Elimination

Immediate Left Recursion
A → Aα₁ | Aα₂ | ... | Aαₘ | β₁ | β₂ | ... | βₙ
Transform to:
A → β₁A' | β₂A' | ... | βₙA'
A' → α₁A' | α₂A' | ... | αₘA' | ε
📌 Example:
Original: E → E + T | T
After: E → T E'
         E' → + T E' | ε

Left Factoring

For Predictive Parsing
A → αβ₁ | αβ₂ | ... | αβₙ
Transform to:
A → αA'
A' → β₁ | β₂ | ... | βₙ
📌 Example:
Original: S → if E then S else S | if E then S
After: S → if E then S S'
        S' → else S | ε

FIRST and FOLLOW Sets

FIRST(α) Computation

FIRST(α) = Set of terminals that begin strings derivable from α
If X is terminal: FIRST(X) = {X}
If X → ε: add ε to FIRST(X)
If X → Y₁Y₂...Yₖ:
Add FIRST(Y₁) - {ε}
If ε ∈ FIRST(Y₁), add FIRST(Y₂) - {ε}
Continue until no ε or end

FOLLOW(A) Computation

FOLLOW(A) = Set of terminals that can appear immediately after A
Add $ to FOLLOW(S) where S is start symbol
If A → αBβ: add FIRST(β) - {ε} to FOLLOW(B)
If A → αB or ε ∈ FIRST(β): add FOLLOW(A) to FOLLOW(B)
📌 Example: FIRST and FOLLOW
Grammar: E → T E', E' → + T E' | ε, T → F T', T' → * F T' | ε, F → ( E ) | id

Non-terminalFIRSTFOLLOW
E( id$ )
E'+ ε$ )
T( id+ $ )
T'* ε+ $ )
F( id* + $ )

LL(1) Parsing

LL(1) Grammar Condition
For productions A → α | β:
1. FIRST(α) ∩ FIRST(β) = ∅
2. If ε ∈ FIRST(α), then FIRST(β) ∩ FOLLOW(A) = ∅

LL(1) Parsing Table Construction

For each production A → α:
  For each terminal a in FIRST(α):
    M[A, a] = A → α
  If ε ∈ FIRST(α):
    For each terminal b in FOLLOW(A):
      M[A, b] = A → α

LL(1) Parsing Algorithm

Initialize: Stack = [$, S], Input = w$
Let X = top of stack, a = current input
If X = a = $: Accept
If X = a ≠ $: Pop X, advance input pointer
If X is non-terminal:
If M[X, a] = error: Report error
Else: Pop X, push RHS of M[X, a] (right to left)

LR Parsing

LR Parser Components

• Stack: States and grammar symbols
• Input: String to parse with $
• Parsing Table: ACTION and GOTO tables
• Output: Handles for semantic analysis

LR Parsing Actions

sn: Shift, push state n
rk: Reduce by production k
acc: Accept input
goto n: Go to state n (after reduce)

LR Parsing Algorithm

Stack = [0], Input = w$, Output = []
Let s = top state, a = current input
If ACTION[s, a] = sn: Push a, push state n, advance input
If ACTION[s, a] = rk (A → β):
Pop 2|β| symbols, let s' = top state
Push A, push GOTO[s', A], output production k
If ACTION[s, a] = acc: Accept
If ACTION[s, a] = error: Report syntax error

Types of LR Parsers

Parser States Grammar Class Complexity
LR(0) Items without lookahead Very restricted Simple
SLR(1) LR(0) items FOLLOW-based reduce Simple, tables small
CLR(1)/LR(1) LR(1) items Most powerful LR Large tables
LALR(1) Merged LR(1) states Between SLR and CLR Most practical

LR Items and Closures

LR(0) Item

Item: Production with dot indicating parsing position
A → α·β (α already seen, β expected)
📌 Example:
E → ·T E' (nothing seen yet)
E → T·E' (T seen, E' expected)
E → T E'· (production complete)

CLOSURE(I)

Add all items in I
If A → α·Bβ in I, add B → ·γ for all B → γ
Repeat until no more items can be added

GOTO(I, X)

GOTO(I, X) = CLOSURE({A → αX·β | A → α·Xβ ∈ I})
Set of items reachable from I by moving past symbol X.

LR(1) Item

Item: [A → α·β, a] where a is lookahead
Lookahead tells us which terminal can follow after reducing.

Conflict Resolution

Shift-Reduce Conflict

Both shift and reduce possible for same state and input
Resolution: Usually prefer shift (correct for most grammars)
⚠️ Example:
Dangling else problem in if-then-else statements

Reduce-Reduce Conflict

Multiple reductions possible for same state and input
Resolution: Usually indicates grammar problem - needs restructuring

🔄 Syntax-Directed Translation

Syntax-Directed Definitions (SDD)

Components
• Grammar: Context-free grammar
• Attributes: Associated values (synthesized, inherited)
• Semantic Rules: Rules for computing attribute values
• Predicates: Conditions that must be satisfied

Synthesized Attributes

Value computed from children nodes
E.val computed from E₁.val and E₂.val
Can be evaluated during bottom-up parsing (post-order traversal).

Inherited Attributes

Value computed from parent/siblings
T.type inherited from parent E.type
Useful for expressing context dependencies (type propagation).

Annotated Parse Tree

📌 Example: Expression Evaluation
ProductionSemantic Rules
E → E₁ + TE.val = E₁.val + T.val
E → E₁ - TE.val = E₁.val - T.val
E → TE.val = T.val
T → T₁ * FT.val = T₁.val * F.val
T → FT.val = F.val
F → ( E )F.val = E.val
F → numF.val = num.lexval
For input: 3 + 4 * 2
E.val = 3 + (4 * 2) = 11

Syntax-Directed Translation Schemes (SDT)

SDT Definition
Grammar with semantic actions embedded in productions
E → E₁ + T { print('+'); }
Action Placement:
• Within production body {action}
• Executed when parser reaches that position during derivation

Postfix Notation SDT

ProductionSemantic Action
E → E₁ + T{ print('+'); }
E → E₁ - T{ print('-'); }
E → T{ }
T → T₁ * F{ print('*'); }
T → T₁ / F{ print('/'); }
T → F{ }
F → num{ print(num.lexval); }
Input: 9 - 5 + 2
Output: 9 5 - 2 + (postfix notation)

Instruction Forms

Abstract Syntax Tree (AST)

AST vs Parse Tree

Parse Tree: Full derivation with all grammar symbols
AST: Compressed representation with only essential nodes
• No parentheses nodes
• No unit productions
• Operators at interior nodes

AST for a = b * c + b * c

        =
       / \
      a   +
         / \
        *   *
       / \ / \
      b  c b  c

Translation of Control Flow

if-else Statement

if (E) S₁ else S₂
ifFalse E goto L1
// code for S₁
goto L2
L1:
// code for S₂
L2:

while Loop

while (E) S
Lbegin:
ifFalse E goto Lend
// code for S
goto Lbegin
Lend:

for Loop

for (E₁; E₂; E₃) S
// code for E₁
Ltest:
ifFalse E₂ goto Lend
// code for S
// code for E₃
goto Ltest
Lend:

Switch Statement

switch(E) { case v₁: S₁; ... }
t = E
if t == v₁ goto L₁
if t == v₂ goto L₂
goto Ldefault
L₁: // code for S₁
goto Lend

Backpatching

One-Pass Code Generation
Used to generate code in one pass when jump targets not yet known.
Makelist(i): Create new list with index i
Merge(p₁, p₂): Merge two lists of jumps
Backpatch(p, i): Set target of all jumps in list p to i
📌 Example: Boolean Expression
Expression: a < b || c < d && e < f
100: if a < b goto _
101: goto 102
102: if c < d goto 104
103: goto _
104: if e < f goto _
105: goto _
Backpatch: 100.truelist, 104.truelist → merge → target filled later

Frame Pointer (FP)
Points to a fixed location in the current activation record. Used to access local variables and parameters.
Stack Pointer (SP)
Points to the top of the stack. Changes as temporaries are pushed/popped during expression evaluation.

Parameter Passing Methods

Method Mechanism Effect
Call by Value Copy value to formal parameter Original unchanged
Call by Reference Pass address of actual parameter Changes affect original
Call by Value-Result Copy in, copy out on return Changes copied back
Call by Name Textual substitution Evaluated when used

Access Links (Static Chain)

Nested Procedures

Access link points to activation record of lexically enclosing procedure
Used to access non-local variables in nested function definitions.

Accessing Non-Local Variables

Follow k access links to reach enclosing scope at depth k
Variable at offset d: FP → access link^k → (FP + d)

Display vs Static Chain

Static Chain

• Each frame has pointer to lexically enclosing frame
• Access requires following chain
• Simple, low overhead on call
• O(n) access for n levels deep

Display

• Array of pointers to most recent activation at each level
• Direct O(1) access to any level
• Higher overhead on call/return
• Better for deeply nested scopes

Types of Optimization

Scope Description Examples

Basic Block

Definition
A basic block is a sequence of consecutive statements with:
• Single entry point (first statement)
• Single exit point (last statement)
• No branching inside the block

Basic Block Construction

Identify leaders (first statements of blocks):
First statement is a leader
Target of conditional/unconditional jump is leader
Statement following jump is leader
Basic block = statements from leader to just before next leader

Common Optimizations

Constant Folding

Evaluate constant expressions at compile time
Before:
x = 3 + 5 * 2
After:
x = 13

Constant Propagation

Replace variable with constant value
Before:
x = 5; y = x + 2
After:
x = 5; y = 7

Common Subexpression Elimination

Compute once, reuse result
Before:
a = b + c
d = b + c
After:
t = b + c
a = t
d = t

Dead Code Elimination

Remove code that doesn't affect program output
Before:
x = 5; x = 10; y = x
After:
x = 10; y = x

Strength Reduction

Replace expensive operations with cheaper ones
Before:
x = y * 2
After:
x = y + y or x = y << 1

Copy Propagation

Use original variable instead of copy
Before:
x = y; z = x + 1
After:
x = y; z = y + 1

Code Motion (Loop Invariant)

Move invariant code outside loop
Before:
for(i=0; i<n; i++) x = y + z;
After:
t = y + z; for(i=0; i<n; i++) x = t;

Induction Variable Elimination

Replace loop counter with derived variables
Before:
for(i=0; i<n; i++) A[i]=0;
After:
for(p=A; p<A+n; p++) *p=0;

Loop Optimizations

Loop Unrolling

Execute multiple iterations in one loop body
Before:
for(i=0; i<4; i++) A[i]=0;
After:
A[0]=0; A[1]=0; A[2]=0; A[3]=0;

Loop Fusion

Combine adjacent loops with same bounds
Before:
for(i=0;i<n;i++) A[i]=0;
for(i=0;i<n;i++) B[i]=0;
After:
for(i=0;i<n;i++) {A[i]=0; B[i]=0;}

Loop Interchange

Swap nested loops for better cache performance
Before:
for(i=0;i<n;i++) for(j=0;j<m;j++) A[i][j]++
After (if row-major):
for(j=0;j<m;j++) for(i=0;i<n;i++) A[i][j]++

Loop Fission/Distribution

Split loop into multiple loops
Useful for parallelization and cache optimization.

Data Flow Analysis

Available Expressions

Expression x+y is available if computed on all paths
OUT[B] = gen[B] ∪ (IN[B] - kill[B])
IN[B] = ∩ OUT[P] for all predecessors P

Live Variables

Variable is live if used later before redefinition
IN[B] = use[B] ∪ (OUT[B] - def[B])
OUT[B] = ∪ IN[S] for all successors S

Reaching Definitions

Definition d reaches point p if path from d to p without redefinition
OUT[B] = gen[B] ∪ (IN[B] - kill[B])
IN[B] = ∪ OUT[P] for all predecessors P

Very Busy Expressions

Expression evaluated on all paths before modification
IN[B] = use[B] ∪ (OUT[B] - kill[B])
OUT[B] = ∩ IN[S] for all successors S

Register Allocation

Graph Coloring Algorithm
Build interference graph: nodes = variables, edges = variables live simultaneously
Try to color graph with k colors (k registers)
If successful: each color = one register
If not: spill some variables to memory

Register Allocation Criteria

Benefit = (uses × memory_cost) - register_cost
Allocate registers to variables with highest usage count (priority).

Spilling

When not enough registers, store variable in memory.
• Load before use
• Store after definition
• Choose variable with longest time until next use

Iterative Algorithm

Worklist Algorithm

Initialize: IN[entry] = ∅, IN[others] = ⊤ (for forward)
Add all blocks to worklist
While worklist not empty:
Remove block B from worklist
Compute new OUT[B] using transfer function
If OUT[B] changed:
Update IN[succ(B)] for all successors
Add successors to worklist

Static Single Assignment (SSA)

SSA Form
Each variable assigned exactly once
φ-functions at join points merge multiple definitions
📌 Example:
// Original
x = 1
if (cond) x = 2
y = x
// SSA Form
x₁ = 1
if (cond) x₂ = 2
x₃ = φ(x₁, x₂)
y = x₃
