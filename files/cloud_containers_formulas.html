Cloud Computing Cost Formulas

TCO (Total Cost of Ownership)

TCO = CAPEX + OPEX + Hidden Costs

CAPEX: Hardware, software licenses, setup

OPEX: Maintenance, power, cooling, staff

Hidden: Downtime, training, migration

Cloud Cost Estimation

Monthly Cost = Compute + Storage + Network + Services

Example AWS EC2 Cost:

t3.medium: $0.0416/hour Г— 730 hours = $30.37/month

+ EBS Storage: $0.10/GB Г— 100GB = $10/month

+ Data Transfer: $0.09/GB Г— 50GB = $4.50/month

Total: ~$45/month

Container Benefits

  • Lightweight: Share host kernel, MBs vs GBs
  • Fast: Start in seconds, not minutes
  • Portable: "Build once, run anywhere"
  • Isolated: Process-level isolation
  • Scalable: Easy to scale horizontally
  • Immutable: Same image in all environments

Dockerfile Structure

Dockerfile Example

# Base image FROM node:18-alpine # Set working directory WORKDIR /app # Copy package files COPY package*.json ./ # Install dependencies RUN npm install # Copy source code COPY . . # Expose port EXPOSE 3000 # Start command CMD ["npm", "start"]

Dockerfile Instructions

InstructionPurpose
FROMBase image
WORKDIRSet working directory
COPYCopy files from host
ADDCopy + extract/URL
RUNExecute command (build)
CMDDefault command (runtime)
ENTRYPOINTExecutable entry point
ENVEnvironment variables
EXPOSEDocument port
VOLUMECreate mount point
ARGBuild-time variables

Docker Commands

Image Commands

$ docker build -t myapp:v1 . $ docker images $ docker pull nginx:latest $ docker push myrepo/myapp:v1 $ docker rmi myapp:v1 $ docker tag myapp:v1 myapp:latest

Container Commands

$ docker run -d -p 80:80 --name web nginx $ docker ps -a $ docker stop web $ docker start web $ docker rm web $ docker exec -it web bash $ docker logs web

Docker Run Options

FlagDescription
-dDetached mode (background)
-p host:containerPort mapping
-v host:containerVolume mount
-e VAR=valueEnvironment variable
--nameContainer name
--networkConnect to network
--restartRestart policy
-m, --memoryMemory limit

Horizontal Pod Autoscaler (HPA)

HPA Formula

desiredReplicas = ceil[currentReplicas Г— (currentMetric / desiredMetric)]

Example:

Current: 2 pods, CPU utilization: 70%

Target: 50% CPU

desiredReplicas = ceil[2 Г— (70/50)] = ceil[2.8] = 3 pods

Lambda Limits & Pricing

Memory 128 MB - 10,240 MB
Timeout 1 sec - 15 min
Cold Start ~100-500ms
Free Tier 1M requests + 400K GB-sec

Lambda Cost Calculation

Cost = (Requests Г— $0.20/1M) + (Duration Г— Memory Г— $0.0000166667/GB-sec)

Example:

1M requests, 512MB, avg 500ms each

Request cost: 1M Г— $0.20/1M = $0.20

Compute: 1M Г— 0.5s Г— 0.5GB Г— $0.0000166667 = $4.17

Total: $4.37/month

CI/CD Best Practices

Continuous Integration

  • Commit often: Small, frequent commits
  • Run tests on every commit: Automated testing
  • Fix broken builds immediately: Priority #1
  • Keep builds fast: Target < 10 minutes
  • Test locally before push: Pre-commit hooks
  • Use feature branches: Branch policies

Continuous Deployment

  • Deploy automatically: After tests pass
  • Use blue-green deployments: Zero downtime
  • Implement canary releases: Gradual rollout
  • Monitor everything: Logs, metrics, traces
  • Rollback quickly: Automated rollback
  • Infrastructure as Code: Terraform, Helm

IAM Best Practices

Identity & Access Management

  • Least Privilege: Grant minimum permissions needed
  • Use Roles: Don't use root/long-term credentials
  • Enable MFA: Multi-factor for all users
  • Rotate Credentials: Regular key rotation
  • Use Groups: Assign permissions to groups, not users
  • Audit: Enable CloudTrail, review regularly

Key Metrics (RED/USE Methods)

RED Method (Request-based)

For services/APIs:

  • Rate - Requests per second
  • Errors - Failed requests per second
  • Duration - Request latency (p50, p95, p99)
Error Budget = 1 - (SLA - Actual Availability)

USE Method (Resource-based)

For infrastructure:

  • Utilization - % time resource busy
  • Saturation - Queue length, backlog
  • Errors - Error events count
Utilization = (Time busy) / (Total time) Г— 100%
