🤖 Finite Automata

DFA (Deterministic Finite Automaton)

Formal Definition
DFA = (Q, Σ, δ, q₀, F)
Where:
• Q = Finite set of states
• Σ = Input alphabet (finite set of symbols)
• δ = Transition function: Q × Σ → Q
• q₀ = Initial/start state (q₀ ∈ Q)
• F = Set of final/accepting states (F ⊆ Q)

Extended Transition Function (δ̂)

δ̂(q, ε) = q
δ̂(q, wa) = δ(δ̂(q, w), a)
Extends δ to process entire strings. δ̂(q₀, w) ∈ F means string w is accepted.

Language Accepted by DFA

L(M) = {w ∈ Σ* | δ̂(q₀, w) ∈ F}
The language accepted by DFA M is the set of all strings that lead from start state to a final state.
📌 Example: DFA for strings ending with 'ab'
Q = {q₀, q₁, q₂}, Σ = {a, b}, F = {q₂}
Stateab
→q₀q₁q₀
q₁q₁q₂
*q₂q₁q₀
→ indicates start state, * indicates final state

NFA (Nondeterministic Finite Automaton)

Formal Definition
NFA = (Q, Σ, δ, q₀, F)
Key Difference from DFA:
• δ = Transition function: Q × (Σ ∪ {ε}) → P(Q)
• P(Q) = Power set of Q (set of all subsets)
• Can have ε-transitions (move without reading input)
• Multiple transitions on same symbol allowed

Extended Transition Function

δ̂(q, ε) = ECLOSE(q)
δ̂(q, wa) = ∪ ECLOSE(δ(r, a)) for all r ∈ δ̂(q, w)
ECLOSE(q) = all states reachable from q using only ε-transitions

Language Accepted by NFA

L(M) = {w ∈ Σ* | δ̂(q₀, w) ∩ F ≠ ∅}
String accepted if ANY path through NFA ends in a final state.
✓ Key Theorem:
DFA and NFA are equivalent in expressive power.
For every NFA, there exists an equivalent DFA (and vice versa).
L(NFA) = L(DFA) - They recognize the same class of languages (Regular Languages).

NFA to DFA Conversion (Subset Construction)

Algorithm

Start state of DFA = ECLOSE(q₀) of NFA
For each DFA state S and input symbol a:
New state = ∪ δ̂(q, a) for all q ∈ S
DFA final states = All states containing any NFA final state
Remove unreachable states
Maximum DFA states = 2^|Q_NFA| (worst case)
📌 Example:
NFA with states {0, 1, 2}
DFA states are subsets: {∅, {0}, {1}, {2}, {0,1}, {0,2}, {1,2}, {0,1,2}}
Maximum 2³ = 8 states (many may be unreachable)

DFA Minimization

Partition Refinement Algorithm

Initial partition: P₀ = {F, Q-F} (final and non-final states)
Refine each group: Split if states go to different groups on same input
Repeat until no further splitting possible
Each final group becomes one state in minimized DFA
💡 Properties:
• Minimum DFA is unique (up to renaming of states)
• Time complexity: O(n² × |Σ|) where n = number of states
• States unreachable from start state should be removed first

Myhill-Nerode Theorem

Indistinguishability Relation
x ≡_L y iff ∀z ∈ Σ*: xz ∈ L ⟺ yz ∈ L
Two strings x and y are indistinguishable with respect to L if for every continuation z, either both xz and yz are in L, or both are not in L.
Theorem Statement
L is regular ⟺ ≡_L has finite index (finite number of equivalence classes)
Number of states in minimum DFA = Number of equivalence classes
Proving Non-Regularity
To prove L is NOT regular: Show infinitely many pairwise distinguishable strings exist.

📝 Regular Expressions

Regular Expression Operations

Operation Notation Description Example
Empty Set ∅ Language with no strings L(∅) = {}
Empty String ε or λ Language containing only empty string L(ε) = {ε}
Symbol a (where a ∈ Σ) Single symbol L(a) = {a}
Concatenation RS or R·S Strings from R followed by strings from S L(ab) = {ab}
Union R + S or R|S or R∪S Strings from R or S L(a+b) = {a, b}
Kleene Star R* Zero or more repetitions L(a*) = {ε, a, aa, aaa, ...}

Algebraic Laws for Regular Expressions

Identity Laws

R + ∅ = R
R · ε = R
ε · R = R

Annihilator

R · ∅ = ∅
∅ · R = ∅

Idempotent Law

R + R = R

Commutative Law

R + S = S + R

Associative Laws

(R + S) + T = R + (S + T)
(RS)T = R(ST)

Distributive Laws

R(S + T) = RS + RT
(R + S)T = RT + ST

Kleene Star Laws

ε* = ε
∅* = ε
R** = R*
R* = ε + R⁺
R⁺ = RR*

Arden's Theorem

If R = Q + RP, then R = QP*
Used to find regular expression from a system of equations. Condition: P must not contain ε.
📌 Example:
If L = aL + b, then L = ba*

DFA/NFA to Regular Expression

State Elimination Method

Add new start state with ε-transition to original start
Add new single final state with ε-transitions from all original final states
Eliminate intermediate states one by one:
For each pair of states i, j with eliminated state k:
New transition from i to j = R_ij + R_ik(R_kk)*R_kj
Result: Single transition from start to final state gives the regular expression

Product Construction for Intersection

DFA₁ = (Q₁, Σ, δ₁, q₁, F₁)
DFA₂ = (Q₂, Σ, δ₂, q₂, F₂)
DFA₁∩₂: Q = Q₁ × Q₂
δ((s₁,s₂), a) = (δ₁(s₁,a), δ₂(s₂,a))
F = F₁ × F₂

Pumping Lemma for Regular Languages

Statement
If L is regular, ∃p > 0 (pumping length) such that:
∀w ∈ L where |w| ≥ p:
w can be divided into xyz where:
1. |y| > 0
2. |xy| ≤ p
3. ∀i ≥ 0: xyⁱz ∈ L

Using Pumping Lemma to Prove Non-Regular

Assume L is regular with pumping length p
Choose a string w ∈ L with |w| ≥ p (careful choice!)
For all possible ways to split w = xyz satisfying conditions:
Find an i such that xyⁱz ∉ L
Contradiction → L is not regular
📌 Example: L = {aⁿbⁿ | n ≥ 0} is NOT regular
Choose w = aᵖbᵖ where p is pumping length
Case 1: y contains only a's → pumping changes number of a's but not b's
Case 2: y contains only b's → similar argument
Case 3: y contains both a's and b's → pumping creates mixed string
All cases lead to contradiction ✓
💡 Important:
Pumping Lemma is a necessary condition, not sufficient.
Some non-regular languages satisfy the pumping lemma (e.g., special cases).

Decision Problems for Regular Languages

Problem Question Decidable? Algorithm
Emptiness L = ∅? Yes Check if any final state reachable from start
Finiteness Is L finite? Yes No cycle on path to final state
Membership w ∈ L? Yes Run DFA on w, check final state
Equivalence L₁ = L₂? Yes (L₁ ∩ L₂') ∪ (L₁' ∩ L₂) = ∅?
Subset L₁ ⊆ L₂? Yes L₁ ∩ L₂' = ∅?

📐 Context-Free Grammars (CFG)

Formal Definition

CFG Definition
G = (V, T, P, S)
Where:
• V = Set of variables (non-terminals)
• T = Set of terminals (Σ - alphabet)
• P = Set of production rules (V → (V ∪ T)*)
• S = Start variable (S ∈ V)

Derivation

α ⇒ β (α derives β in one step)
α ⇒* β (α derives β in zero or more steps)
α ⇒⁺ β (α derives β in one or more steps)
Derivation: Replace a variable using a production rule.

Language Generated

L(G) = {w ∈ T* | S ⇒* w}
The language generated by G consists of all strings of terminals that can be derived from the start symbol.
📌 Example: CFG for {aⁿbⁿ | n ≥ 0}
G = ({S}, {a,b}, P, S)
P: S → aSb | ε
Derivation of a³b³: S ⇒ aSb ⇒ aaSbb ⇒ aaaSbbb ⇒ aaabbb
This grammar generates strings with equal numbers of a's followed by b's.

Derivation Trees (Parse Trees)

Properties

Root labeled with start symbol S
Leaves labeled with terminals or ε
Internal nodes labeled with variables
Children of node A spell out right side of production A → α

Leftmost vs Rightmost Derivation

• Leftmost: Always replace leftmost variable first
• Rightmost: Always replace rightmost variable first
• Both give same parse tree for unambiguous grammar

Ambiguity

Definition
A CFG G is ambiguous if ∃w ∈ L(G) that has more than one:
• Parse tree, OR
• Leftmost derivation, OR
• Rightmost derivation
📌 Example: Ambiguous Grammar for Expressions
E → E + E | E * E | id
String "id + id * id" has two parse trees:
• (id + id) * id - multiplication first
• id + (id * id) - addition first
Unambiguous version:
E → E + T | T
T → T * F | F
F → id | (E)

Chomsky Normal Form (CNF)

Definition
CNF Rules: A → BC or A → a
• Only productions allowed: A → BC (two variables) or A → a (single terminal)
• Start symbol S can also produce ε: S → ε (if ε ∈ L)
• No unit productions (A → B) or ε-productions (A → ε, except S → ε)

Conversion to CNF

Add new start symbol S₀ → S (if S appears on RHS)
Remove ε-productions (except S → ε): For A → αBβ where B is nullable, add A → αβ
Remove unit productions A → B: If B → α, add A → α
Replace terminals in RHS of length > 1: A → aB becomes A → XₐB, Xₐ → a
Break long RHS: A → BCD becomes A → BE, E → CD

Greibach Normal Form (GNF)

Definition
GNF Rules: A → aα where a ∈ T, α ∈ V* (zero or more variables)
Every production starts with a terminal followed by zero or more variables.
📌 Example:
S → aABC, A → bB, B → c, C → d

📚 Pushdown Automata (PDA)

Formal Definition

PDA Definition
PDA = (Q, Σ, Γ, δ, q₀, Z₀, F)
Where:
• Q = Finite set of states
• Σ = Input alphabet
• Γ = Stack alphabet
• δ = Transition: Q × (Σ ∪ {ε}) × Γ → P(Q × Γ*)
• q₀ = Initial state
• Z₀ = Initial stack symbol
• F = Set of final states

Acceptance by Final State

L(M) = {w | (q₀, w, Z₀) ⊢* (q, ε, γ), q ∈ F}
String accepted if PDA reaches a final state after consuming all input.

Acceptance by Empty Stack

L(M) = {w | (q₀, w, Z₀) ⊢* (q, ε, ε)}
String accepted if stack becomes empty after consuming all input.
✓ Equivalence:
Both acceptance methods are equivalent. A PDA accepting by final state can be converted to one accepting by empty stack, and vice versa.

PDA for Context-Free Languages

📌 Example: PDA for {aⁿbⁿ | n ≥ 0}
States: Q = {q₀, q₁, q₂}
Stack: Γ = {Z, A}
StateInputStack TopOperationNext State
q₀aZpush Aq₀
q₀aApush Aq₀
q₀bApopq₁
q₁bApopq₁
q₁εZ—q₂
Algorithm: Push A for each a, pop A for each b, accept if stack empty after all b's

Equivalence: CFG ↔ PDA

CFG → PDA Conversion

Single state q
For each A → α in P: add transition that pops A and pushes α
For each terminal a: add transition that pops a when reading a
Start with S on stack

PDA → CFG Conversion

Variables: Aₚq for states p, q
Aₚq generates strings taking PDA from p to q with empty net stack change
For δ(p, a, X) contains (r, Y₁Y₂...Yₖ):
Add Aₚq → aAr₁Ar₂...Arₖ₋₁q (for all q)

Pumping Lemma for CFLs

Statement
If L is a CFL, ∃p > 0 (pumping length) such that:
∀w ∈ L where |w| ≥ p:
w can be divided into uvxyz where:
1. |vxy| ≤ p
2. |vy| > 0
3. ∀i ≥ 0: uvⁱxyⁱz ∈ L
📌 Example: L = {aⁿbⁿcⁿ | n ≥ 0} is NOT a CFL
Choose w = aᵖbᵖcᵖ
Since |vxy| ≤ p, vxy cannot contain all three symbols
Pumping v and y will increase count of some symbols but not all three equally
Result: uv²xy²z ∉ L → Contradiction ✓

CYK Algorithm (Membership Testing)

Algorithm for CFG in CNF

Input: String w = w₁w₂...wₙ and CFG G in CNF
Create n × n table T
For i = 1 to n: T[i,i] = {A | A → wᵢ in P}
For length l = 2 to n:
For i = 1 to n-l+1:
j = i + l - 1
T[i,j] = {A | A → BC, B ∈ T[i,k], C ∈ T[k+1,j] for some k}
w ∈ L(G) iff S ∈ T[1,n]
Time Complexity: O(n³ × |P|)
Space Complexity: O(n²)

🖥️ Turing Machines

Formal Definition

TM Definition
TM = (Q, Σ, Γ, δ, q₀, B, F)
Where:
• Q = Finite set of states
• Σ = Input alphabet (⊂ Γ, excluding B)
• Γ = Tape alphabet (includes blank symbol B)
• δ = Transition: Q × Γ → Q × Γ × {L, R, S}
• q₀ = Initial state
• B = Blank symbol (B ∈ Γ)
• F = Set of final/halting states

Instantaneous Description (ID)

ID: αqβ where:
• α = tape contents left of head
• q = current state
• β = tape contents at and right of head
📌 Example:
1011q₅01 represents: tape has 101101, head at 0, state q₅

Move Relation (⊢)

If δ(q, X) = (p, Y, R):
αqXβ ⊢ αYpβ
If δ(q, X) = (p, Y, L):
ZaXqβ ⊢ ZpYaβ (if Y ≠ B)

Language Classes

Recursively Enumerable (RE)

L is RE ⟺ ∃TM M such that L = L(M)
TM accepts all strings in L (may loop on strings not in L).
L(M) = {w | q₀w ⊢* αqₐβ for some qₐ ∈ F}

Recursive (Decidable)

L is Recursive ⟺ ∃TM M that halts on all inputs and L = L(M)
TM accepts strings in L and rejects (halts) strings not in L.
✓ Relationship:
Recursive ⊂ RE ⊂ All Languages
• Recursive: TM always halts
• RE: TM may loop on strings not in L
• Co-RE: Complement is RE

TM Variants

Variant Feature Equivalent to Standard TM?
Multi-tape TM k tapes with independent heads Yes
Non-deterministic TM Multiple possible moves Yes
Two-way infinite tape Tape extends both directions Yes
Multi-dimensional tape 2D or higher tape Yes
Enumerator Outputs strings of L Yes
💡 Church-Turing Thesis:
Any computation that can be carried out by a mechanical procedure can be performed by a Turing Machine. All reasonable computational models are equivalent.

Encoding and Universal TM

Universal Turing Machine U
U takes ⟨M, w⟩ as input and simulates M on w
• ⟨M, w⟩ is an encoding of TM M and input w
• U is a programmable TM that can simulate any other TM
• Foundation of stored-program computers

❓ Undecidability

Key Undecidable Problems

Halting Problem
HALT = {⟨M, w⟩ | M halts on input w}
Given a TM M and input w, determine if M halts on w.
Proof: Diagonalization - assume decider H exists, construct D that contradicts
Undecidable:
No algorithm can determine if an arbitrary program halts on given input.

Other Undecidable Problems

A_TM = {⟨M, w⟩ | M accepts w}
EMPTY_TM = {⟨M⟩ | L(M) = ∅}
EQ_TM = {⟨M₁, M₂⟩ | L(M₁) = L(M₂)}
ALL_TM = {⟨M⟩ | L(M) = Σ*}
REGULAR_TM = {⟨M⟩ | L(M) is regular}

Rice's Theorem

Every non-trivial property of RE languages is undecidable.
Non-trivial property: Some RE languages have it, some don't.
Examples: "L is finite", "L contains ε", "L is regular"

Reduction

Mapping Reduction
A ≤ₘ B (A reduces to B) if ∃ computable f: w ∈ A ⟺ f(w) ∈ B
If A ≤ₘ B and B is decidable, then A is decidable.
If A ≤ₘ B and A is undecidable, then B is undecidable.

Using Reductions

To prove B is undecidable:
Find known undecidable problem A
Show A ≤ₘ B (construct reduction)
If A were decidable through B, contradiction
Common reductions: HALT → A_TM → EMPTY_TM → EQ_TM

Post Correspondence Problem (PCP)

Definition
PCP: Given dominos (x₁,y₁), (x₂,y₂), ..., (xₖ,yₖ)
Find sequence i₁i₂...iₘ such that xᵢ₁xᵢ₂...xᵢₘ = yᵢ₁yᵢ₂...yᵢₘ
PCP is undecidable. Used to prove many problems in formal languages undecidable.

⏱️ Complexity Theory

Time and Space Complexity

Definitions
TIME(f(n)) = {L | ∃TM decides L in O(f(n)) time}
SPACE(f(n)) = {L | ∃TM decides L in O(f(n)) space}
• n = input length
• Time = number of steps
• Space = number of tape cells used

Complexity Classes

Class P

P = ∪ₖ TIME(nᵏ)
Languages decidable in polynomial time by a deterministic TM.
"Efficiently solvable" problems.
📌 Examples in P:
Path finding, sorting, matrix multiplication, primality testing

Class NP

NP = {L | ∃ polynomial-time verifier for L}
Languages verifiable in polynomial time.
"Efficiently checkable" problems.
📌 Examples in NP:
SAT, Hamiltonian path, graph coloring, subset sum

Class PSPACE

PSPACE = ∪ₖ SPACE(nᵏ)
Languages decidable in polynomial space.

Class EXPTIME

EXPTIME = ∪ₖ TIME(2^(nᵏ))
Languages decidable in exponential time.

Class co-NP

co-NP = {L̄ | L ∈ NP}
Complements of NP languages.

Class NL

NL = NSPACE(log n)
Nondeterministic logarithmic space.

Hierarchy of Classes

    P ⊆ NP ⊆ PSPACE ⊆ EXPTIME ⊆ NEXPTIME
    
    Known strict inclusions:
    P ⊂ EXPTIME
    
    Unknown (major open problems):
    P vs NP (P = NP?)
    NP vs co-NP
    P vs PSPACE

NP-Completeness

Definitions
L is NP-Hard: ∀L' ∈ NP: L' ≤ₚ L (polynomial-time reduction)
L is NP-Complete: L ∈ NP AND L is NP-Hard

Proving NP-Completeness

Show L ∈ NP (give polynomial-time verifier)
Choose known NP-Complete problem L'
Show L' ≤ₚ L (polynomial-time reduction)
Conclude L is NP-Complete
📌 First NP-Complete Problem: SAT
SAT (Boolean Satisfiability): Given boolean formula φ, is there an assignment making φ true?
Cook-Levin Theorem: SAT is NP-Complete

Space Complexity Theorems

Savitch's Theorem

NSPACE(f(n)) ⊆ SPACE(f(n)²)
Nondeterministic space can be simulated by deterministic space with quadratic overhead.
Corollary: NPSPACE = PSPACE

Immerman-Szelepcsényi Theorem

NSPACE(f(n)) = co-NSPACE(f(n))
Nondeterministic space classes are closed under complement (for f(n) ≥ log n).
